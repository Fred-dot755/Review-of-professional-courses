# 第3章 存储系统

> 存储系统是计算机中用来存放程序和数据的核心子系统。现代计算机的存储系统采用层次化结构，利用程序访问的局部性原理，以合理的成本实现近乎最快的速度和近乎最大的容量。本章是计算机组成原理的重中之重。

---

## 3.1 存储系统概述

### 3.1.1 存储器的分类

#### 按存储介质分类

| 类型 | 介质 | 特点 | 典型实例 |
|------|------|------|----------|
| **半导体存储器** | 半导体器件 | 速度快、体积小 | 内存（DRAM）、Cache（SRAM） |
| **磁表面存储器** | 磁性材料 | 非易失、容量大、速度慢 | 硬盘（HDD） |
| **光盘存储器** | 光介质 | 可移动、寿命长 | CD、DVD、Blu-ray |
| **闪存（Flash）** | 半导体 | 非易失、速度快于磁盘 | SSD、U盘 |

#### 按存取方式分类

| 类型 | 存取方式 | 特点 |
|------|----------|------|
| **RAM**（随机存取存储器） | 按地址访问，存取时间与物理位置无关 | 易失性 |
| **ROM**（只读存储器） | 正常工作只能读 | 非易失，掉电不丢失 |
| **SAM**（顺序存取存储器） | 按顺序访问 | 磁带 |
| **DAM**（直接存取存储器） | 先找到区域再顺序查找 | 磁盘 |

#### 按在计算机中的作用分类

```
┌──────────────────────────────────────────────────────┐
│  CPU内部                                            │
│   ┌──────────┐                                       │
│   │  寄存器   │  ← 速度最快，容量最小（几十~几百字节）   │
│   └──────────┘                                       │
├──────────────────────────────────────────────────────┤
│   CPU内部或紧邻                                      │
│   ┌──────────┐                                       │
│   │  Cache   │  ← SRAM实现，速度接近CPU               │
│   └──────────┘                                       │
├──────────────────────────────────────────────────────┤
│   主板上                                            │
│   ┌──────────┐                                       │
│   │  主存     │  ← DRAM实现，程序运行的场所            │
│   └──────────┘                                       │
├──────────────────────────────────────────────────────┤
│   外部设备                                          │
│   ┌──────────┐                                       │
│   │  辅存     │  ← 磁盘、SSD、光盘等                  │
│   └──────────┘                                       │
└──────────────────────────────────────────────────────┘
```

### 3.1.2 存储系统的层次结构 ⭐

```
         ┌─────┐
         │CPU  │
      ┌──┤寄存器├──┐
      │  └─────┘  │
      │ 速度最快   │
      │ 容量最小   │
      ▼ 价格最高   ▼
   ┌─────────┐  ┌─────────┐
   │ L1 Cache│  │ L1 Cache│  Cache层
   │ (指令)  │  │ (数据)  │  (SRAM)
   └────┬────┘  └────┬────┘
        └──────┬─────┘
               ▼
        ┌────────────┐
        │  L2 Cache   │
        └──────┬─────┘
               ▼
        ┌────────────┐
        │  L3 Cache   │  (现代CPU常见)
        └──────┬─────┘
               ▼
        ┌────────────┐
        │  主存(DRAM) │  主存层
        └──────┬─────┘
               ▼
        ┌────────────┐
        │  辅存(磁盘) │  辅存层
        └────────────┘
        速度最慢 容量最大 价格最低
```

#### 层次结构的关键特性

| 特性 | 上层（近CPU）→ 下层（远CPU） |
|------|------------------------------|
| 速度 | 越来越慢 |
| 容量 | 越来越大 |
| 每位价格 | 越来越低 |
| CPU访问频率 | 越来越低 |

> **核心原则**：层次结构之所以有效，依赖于**程序访问的局部性原理**（Principle of Locality）。

### 3.1.3 程序访问的局部性原理

| 类型 | 含义 | 典型示例 |
|------|------|----------|
| **时间局部性**（Temporal Locality） | 最近访问过的数据/指令，近期很可能再次被访问 | 循环中的变量、频繁调用的子程序 |
| **空间局部性**（Spatial Locality） | 即将访问的数据/指令很可能在最近访问过的数据/指令附近 | 顺序执行的指令、数组元素 |

> 正是由于局部性原理，我们才能用较小的Cache捕获大部分的内存访问，使平均访问时间接近Cache的访问时间。

## 3.2 主存储器（Main Memory）

### 3.2.1 主存的基本结构

主存由**存储体**（存储单元的集合）、**地址译码器**、**读写电路**和控制逻辑组成。

```
地址总线 ──▶ ┌──────────┐
             │ 地址寄存器 │──▶ 地址译码器 ──▶ 存储体（存储矩阵）
             └──────────┘                         │
                                                  ▼
数据总线 ◀──▶ ┌──────────┐                 ┌──────────┐
             │ 数据寄存器 │◀───────────────▶│ 读写电路  │
             └──────────┘                 └──────────┘
                                                  ▲
控制总线 ──▶ ┌──────────┐                         │
             │ 控制逻辑  │─────────────────────────┘
             └──────────┘
```

### 3.2.2 主存的主要性能指标

| 指标 | 含义 | 典型值 |
|------|------|--------|
| **存储容量** | 可存储的二进制信息总量 | 8GB、16GB、32GB |
| **存取时间**（Access Time） | 从发出地址到完成读/写的时间 | DDR4约10~20ns |
| **存储周期**（Memory Cycle） | 两次独立访问的最小间隔时间 | 略大于存取时间 |
| **带宽**（Bandwidth） | 单位时间传输的数据量 | 几十GB/s |

> **存取时间 ≠ 存储周期**：存取时间是完成一次读/写的时间，存储周期还需要加上恢复时间。

### 3.2.3 半导体存储器芯片

#### SRAM（静态随机存取存储器）

| 特征 | 说明 |
|------|------|
| 存储单元 | 双稳态触发器（6个MOS管） |
| 特点 | 速度快、功耗低（待机时）、集成度低、价格高 |
| 用途 | Cache（高速缓存） |
| 是否需要刷新 | ❌ 不需要（只要有电就能保持） |

#### DRAM（动态随机存取存储器）

| 特征 | 说明 |
|------|------|
| 存储单元 | 电容（1个MOS管+1个电容） |
| 特点 | 集成度高、价格低、速度慢、功耗大（工作时） |
| 用途 | 主存（内存条） |
| 是否需要刷新 | ✅ 必须定期刷新（电容会漏电） |

#### 刷新方式

| 刷新方式 | 说明 |
|----------|------|
| **集中刷新** | 在固定时间段内集中刷新所有行，此时无法读写（存在"死区"） |
| **分散刷新** | 将刷新操作分散到各工作周期中，无死区但系统速度降低 |
| **异步刷新** | 在固定刷新周期内，将各行刷新均匀分布（折中方案，最常用） |

### 3.2.4 ROM（只读存储器）

| 类型 | 全称 | 特点 |
|------|------|------|
| **MROM** | Mask ROM | 出厂时掩模写入，不可更改 |
| **PROM** | Programmable ROM | 用户可一次性写入 |
| **EPROM** | Erasable PROM | 紫外线擦除后可重写 |
| **EEPROM** | Electrically EPROM | 电擦除、可重写 |
| **Flash Memory** | 闪存 | 电擦除、块擦除、速度快，SSD和U盘的核心 |

## 3.3 主存与CPU的连接

### 3.3.1 存储容量的扩展

| 扩展方式 | 方法 | 解决的问题 |
|----------|------|-----------|
| **位扩展** | 多片芯片并联，共用地址线和控制线 | 数据位数不足 |
| **字扩展** | 增加地址线，通过片选信号选择芯片 | 存储单元数不足 |
| **字位同时扩展** | 先位扩展成组，再对组进行字扩展 | 同时增加位数和容量 |

### 3.3.2 地址译码方式

| 方式 | 原理 | 特点 |
|------|------|------|
| **线选法** | 单根高位地址线直接连接一个芯片的片选端 | 简单但地址不连续，有地址重叠 |
| **译码片选法** | 高位地址经过译码器产生片选信号 | 地址连续无重叠，但译码器有延迟 |

## 3.4 Cache（高速缓冲存储器）⭐

> Cache是存储系统章节中最核心的内容。理解Cache的映射方式、替换算法和写策略是掌握现代计算机存储体系的关键。

### 3.4.1 Cache的工作原理

#### 基本概念

CPU访问数据时，**先查Cache，如果命中（Hit）则直接访问，如果不命中（Miss）则访问主存并将该数据块调入Cache**。

#### 命中率与平均访问时间

```
命中率 h = Cache命中次数 / 总访问次数

平均访问时间 Ta = h × Tc + (1-h) × Tm
```

其中 Tc 为 Cache 访问时间，Tm 为主存访问时间。

#### 访问效率

```
e = Tc / Ta × 100%
```

### 3.4.2 Cache的地址映射方式 ⭐⭐⭐

#### 全相联映射（Fully Associative）

```
主存中的任意一块可以映射到Cache中的任意一块。

优点：冲突概率最低，Cache利用率最高
缺点：比较电路复杂，成本高，速度慢
适用：小容量Cache
```

#### 直接映射（Direct Mapped）

```
主存中的每一块只能映射到Cache中的唯一一块。

Cache行号 = 主存块号 mod Cache总行数

优点：硬件简单，比较速度快，成本低
缺点：冲突概率高（抖动），Cache利用率低
适用：大容量Cache
```

#### 组相联映射（Set Associative）⭐

```
Cache被分成多个组，每组包含多行。
主存块先确定映射到哪一组，然后可以放到该组中的任意一行。

Cache组号 = 主存块号 mod Cache组数

这是全相联和直接映射的折中方案，现代CPU广泛使用（如8路组相联）。
```

| 映射方式 | 冲突概率 | 硬件复杂度 | 命中率 | 应用 |
|----------|----------|------------|--------|------|
| 全相联 | 最低 | 最高 | 最高 | 小容量TLB |
| 直接映射 | 最高 | 最低 | 最低 | 早期设计 |
| **组相联** | **中等** | **中等** | **较高** | **现代CPU主力** |

### 3.4.3 替换算法

当Cache满而需要调入新块时，必须选择一块替换出去。

| 算法 | 英文 | 原理 | 特点 |
|------|------|------|------|
| **随机替换**（RAND） | Random | 随机选择被替换的块 | 简单但不理想 |
| **先进先出**（FIFO） | First In First Out | 替换最早调入的块 | 不符合局部性原理 |
| **近期最少使用**（LRU）⭐ | Least Recently Used | 替换最久未被访问的块 | 符合局部性，命中率高，但硬件较复杂 |
| **最不经常使用**（LFU） | Least Frequently Used | 替换访问次数最少的块 | 需要计数器 |

> **LRU是最常用的替换算法**。对于2路组相联，实现LRU只需要1位标志；对于4路组相联，需要跟踪6种可能的访问顺序关系。

### 3.4.4 写策略（Write Policy）

这是Cache设计中至关重要的问题。当CPU执行写操作时：

#### 写命中（Write Hit）时

| 策略 | 英文 | 原理 | 特点 |
|------|------|------|------|
| **写直达（全写法）** | Write Through | 同时写入Cache和主存 | 简单，但写操作速度慢，总线流量大 |
| **写回法** ⭐ | Write Back | 只写Cache，被替换时才写回主存 | 速度快，但需要脏位（Dirty Bit），一致性管理复杂 |

#### 写不命中（Write Miss）时

| 策略 | 英文 | 原理 |
|------|------|------|
| **写分配法** | Write Allocate | 先将块调入Cache，再写入（通常和写回法搭配） |
| **非写分配法** | No-Write Allocate | 直接写主存，不调入Cache（通常和写直达法搭配） |

```
常见组合：
- 写回法 + 写分配法（现代CPU主流）
- 写直达法 + 非写分配法
```

#### Cache一致性问题

在多核处理器或DMA场景下，多个设备可能同时访问同一块数据，需要维护一致性：

| 协议 | 全称 | 特点 |
|------|------|------|
| **MESI协议** | Modified, Exclusive, Shared, Invalid | 最经典的Cache一致性协议 |
| **MOESI协议** | 增加Owned状态 | AMD使用的增强版 |
| **目录协议** | Directory-based | 适合大规模多处理器系统 |

### 3.4.5 Cache性能优化

| 技术 | 原理 |
|------|------|
| **多级Cache** | L1小而快，L2/L3大而稍慢，分级覆盖 |
| **分离Cache** | 指令Cache和数据Cache分开（哈佛结构） |
| **预取（Prefetch）** | 硬件或软件提前将可能需要的数据调入Cache |
| **Victim Cache** | 被替换出的数据暂存于小容量Cache中，减少冲突缺失 |

## 3.5 虚拟存储器（Virtual Memory）⭐

### 3.5.1 虚拟存储器的基本概念

虚拟存储器将主存和外存（磁盘/SSD）统一编址，形成一个逻辑上远大于物理内存的地址空间。

#### 三个地址空间

| 地址 | 含义 |
|------|------|
| **逻辑地址（虚拟地址）** | 程序员看到的地址，CPU发出的地址 |
| **物理地址（实地址）** | 主存的实际地址 |
| **辅存地址** | 磁盘上的地址（虚存映射到磁盘的部分） |

### 3.5.2 页式虚拟存储器 ⭐

将虚拟地址空间和物理地址空间都划分为**固定大小的页（Page）**，以页为单位进行映射。

```
虚拟地址：  虚拟页号（VPN）  │  页内偏移（Offset）
                              │
              映射（页表）     │
                              ▼
物理地址：  物理页号（PPN）  │  页内偏移（Offset）
```

#### 页表（Page Table）

页表是虚拟页号到物理页号的映射表，每个进程都有自己的页表。

| 页表项字段 | 含义 |
|-----------|------|
| **物理页号** | 映射到的物理页框号 |
| **有效位（Valid Bit）** | 该页是否在主存中（1=在，0=缺页） |
| **修改位（Dirty Bit）** | 该页是否被修改过（用于写回策略） |
| **引用位（Reference Bit）** | 该页最近是否被访问过（用于替换算法） |
| **访问权限位** | 读/写/执行权限 |

### 3.5.3 TLB（快表）⭐

> **TLB（Translation Lookaside Buffer）** 是页表的高速缓存，用于加速虚拟地址到物理地址的转换。

```
CPU发出虚拟地址
      │
      ▼
┌─────────────┐
│  查询 TLB   │ ← 命中：直接获得物理页号（快）
└─────┬───────┘
      │ 未命中
      ▼
┌─────────────┐
│  查询 页表   │ ← 访问主存中的页表（慢）
└─────┬───────┘
      │
      ▼
┌─────────────┐
│  更新 TLB   │
└─────┬───────┘
      ▼
获得物理地址，访问Cache/主存
```

- TLB通常采用**全相联**或**组相联**映射（容量小，追求高命中率）
- TLB缺失比Cache缺失的代价大得多，因为需要访问主存中的页表

### 3.5.4 段式与段页式虚拟存储器

| 类型 | 划分方式 | 优点 | 缺点 |
|------|----------|------|------|
| **页式** | 固定大小的页 | 管理简单，无外碎片 | 存在页内碎片 |
| **段式** | 按逻辑意义划分（代码段、数据段等） | 符合程序逻辑，便于共享和保护 | 存在外碎片 |
| **段页式** | 先分段，每段内再分页 | 兼具两者优点 | 地址转换开销大 |

### 3.5.5 页面替换算法

当主存满且需要调入新页时，选择一页调出：

| 算法 | 原理 | 特点 |
|------|------|------|
| **OPT**（最佳置换） | 淘汰将来最长时间不用的页 | 理论最优但无法实现，作为比较基准 |
| **FIFO** | 淘汰最先调入的页 | Belady异常（增加页面数反而缺页增多） |
| **LRU**（最近最久未使用）⭐ | 淘汰最久未被访问的页 | 接近OPT，性能好但实现成本高 |
| **Clock**（时钟算法） | 近似LRU，循环检查访问位 | 硬件支持简单，实际系统常用（如Linux） |

## 3.6 辅助存储器

### 3.6.1 硬盘（HDD）

| 指标 | 说明 |
|------|------|
| **容量** | 面数 × 磁道数 × 扇区数 × 扇区容量 |
| **转速** | 5400rpm, 7200rpm, 10000rpm, 15000rpm |
| **平均存取时间** | 寻道时间 + 旋转延迟 + 数据传输时间 |
| **平均旋转延迟** | 旋转半圈的时间 = 1/(2 × 转速) |

### 3.6.2 固态硬盘（SSD）

| 特征 | 对比HDD |
|------|---------|
| 介质 | NAND Flash vs 磁介质 |
| 随机读写 | 远快于HDD（无机械部件） |
| 寿命 | 有写入次数限制（P/E Cycles） |
| 价格 | 每GB价格高于HDD |

## 本章小结

存储系统是计算机组成原理中最核心的章节，核心知识体系如下：

```
                   存储系统层次结构
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
      Cache层          主存层          虚拟存储
         │               │               │
    ┌────┴────┐     ┌────┴────┐     ┌────┴────┐
    │映射方式  │     │DRAM/SRAM│     │页式/段式 │
    │替换算法  │     │容量扩展  │     │页表/TLB  │
    │写策略   │     │刷新机制  │     │替换算法  │
    └─────────┘     └─────────┘     └─────────┘
```

**最重要的几个核心概念**：
1. **局部性原理**是层次存储系统赖以工作的基础
2. **Cache的组相联映射 + LRU替换 + 写回法**是现代CPU的主流设计
3. **TLB + 页表**构成虚拟存储器的核心地址转换机制
4. 理解这些概念对写出高性能的程序至关重要（如避免Cache伪共享、利用数据局部性等）
