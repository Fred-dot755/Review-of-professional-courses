# 第2章 分治法

## 2.1 分治法基本思想

### 2.1.1 分治法的三个步骤

分治法（Divide and Conquer）将一个规模为 n 的问题分解为 k 个规模较小的子问题，这些子问题相互独立且与原问题相同。递归解决子问题，再合并结果。

```
┌──────────────────────────────────────────────────┐
│  分治法三步骤：                                    │
│  ① 分解（Divide）：将原问题分解为若干子问题         │
│  ② 解决（Conquer）：递归求解子问题（到足够小则直接解）│
│  ③ 合并（Combine）：将子问题的解合并为原问题的解     │
└──────────────────────────────────────────────────┘
```

### 2.1.2 分治法的适用条件

1. 问题规模缩小到一定程度可以容易解决
2. 问题可以分解为若干个相同类型的子问题（**最优子结构**）
3. 子问题的解可以合并为原问题的解
4. 子问题之间**相互独立**（不包含公共子问题）—— 这是与动态规划的关键区别

### 2.1.3 分治法的时间复杂度

设分解和合并的总代价为 f(n)，则：

> T(n) = k·T(n/m) + f(n)

其中 m 为子问题规模的缩小因子，k 为子问题个数。可用 Master 定理求解。

## 2.2 二分查找（Binary Search）

### 2.2.1 算法思想

在**有序**序列中，每次将查找区间折半，比较中间元素与目标值，决定在左半区还是右半区继续查找。

### 2.2.2 算法实现

```c
int BinarySearch(int a[], int n, int x) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = (left + right) / 2;
        if (a[mid] == x) return mid;
        else if (a[mid] < x) left = mid + 1;
        else right = mid - 1;
    }
    return -1;  // 未找到
}
```

### 2.2.3 复杂度分析

每次比较将问题规模缩小一半：T(n) = T(n/2) + O(1)

由 Master 定理：**T(n) = O(log n)**

## 2.3 归并排序（Merge Sort）

### 2.3.1 算法思想

1. **分解**：将待排序序列分成两个等长的子序列
2. **解决**：递归地对两个子序列进行归并排序
3. **合并**：将两个有序子序列合并为一个有序序列

### 2.3.2 算法实现

```c
// 合并两个有序子数组
void Merge(int a[], int left, int mid, int right) {
    int i = left, j = mid + 1, k = 0;
    int *temp = (int*)malloc((right - left + 1) * sizeof(int));
    
    while (i <= mid && j <= right) {
        if (a[i] <= a[j]) temp[k++] = a[i++];
        else temp[k++] = a[j++];
    }
    while (i <= mid) temp[k++] = a[i++];
    while (j <= right) temp[k++] = a[j++];
    
    for (i = 0; i < k; i++)
        a[left + i] = temp[i];
    free(temp);
}

// 归并排序
void MergeSort(int a[], int left, int right) {
    if (left < right) {
        int mid = (left + right) / 2;
        MergeSort(a, left, mid);      // 排序左半
        MergeSort(a, mid + 1, right);  // 排序右半
        Merge(a, left, mid, right);    // 合并
    }
}
```

### 2.3.3 复杂度分析

- **递归方程**：T(n) = 2T(n/2) + O(n)（合并 O(n)，分解 O(1)）
- Master 定理（情况2）：**T(n) = Θ(n log n)**
- **空间复杂度**：O(n)（需要辅助数组）

### 2.3.4 特点总结

| 优点 | 缺点 |
|------|------|
| 稳定排序 | 需要 O(n) 额外空间 |
| 时间复杂度始终 O(n log n) | 递归调用有开销 |
| 适合外部排序 | 对小规模数据不如插入排序 |

## 2.4 快速排序（Quick Sort）

### 2.4.1 算法思想

1. **分解**：选取一个**基准元素**（pivot），将序列分为左右两部分：左边 ≤ pivot ≤ 右边
2. **解决**：递归地对左右两部分进行快速排序
3. **合并**：不需要显式合并，分区后自然有序

### 2.4.2 算法实现

```c
// 一趟划分
int Partition(int a[], int left, int right) {
    int pivot = a[left];  // 选取第一个元素为基准
    while (left < right) {
        while (left < right && a[right] >= pivot) right--;
        a[left] = a[right];
        while (left < right && a[left] <= pivot) left++;
        a[right] = a[left];
    }
    a[left] = pivot;
    return left;  // 返回基准元素最终位置
}

// 快速排序
void QuickSort(int a[], int left, int right) {
    if (left < right) {
        int pivotPos = Partition(a, left, right);
        QuickSort(a, left, pivotPos - 1);
        QuickSort(a, pivotPos + 1, right);
    }
}
```

### 2.4.3 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好情况 | O(n log n) | 每次基准恰好将序列等分 |
| 平均情况 | O(n log n) | 期望分区比较平衡 |
| 最坏情况 | O(n²) | 序列已有序，每次只减少1个元素 |

最坏递归方程：T(n) = T(n-1) + O(n) ⇒ T(n) = O(n²)

平均递归方程：T(n) = 2T(n/2) + O(n) ⇒ T(n) = O(n log n)

**空间复杂度**：最好 O(log n)（递归栈深度），最坏 O(n)

### 2.4.4 快速排序 vs 归并排序

| 维度 | 快速排序 | 归并排序 |
|------|---------|---------|
| 时间（平均） | O(n log n) | O(n log n) |
| 时间（最坏） | O(n²) | O(n log n) |
| 空间 | O(log n) | O(n) |
| 稳定性 | ❌ 不稳定 | ✅ 稳定 |
| 实际速度 | 通常更快（常数因子小） | 较慢 |

## 2.5 大整数乘法（Karatsuba算法）

### 2.5.1 问题描述

两个 n 位整数相乘，传统乘法需要 O(n²) 次位运算。

### 2.5.2 分治策略

将两个 n 位整数 X, Y 各分成两段：

```
X = A × 10^(n/2) + B
Y = C × 10^(n/2) + D
```

则：
```
X × Y = AC × 10ⁿ + (AD + BC) × 10^(n/2) + BD
```

需要计算 4 次 n/2 位的乘法：AC, AD, BC, BD

**Karatsuba 优化**：利用公式，只需 **3 次** n/2 位乘法：

```
X × Y = AC × 10ⁿ + ((A+B)(C+D) - AC - BD) × 10^(n/2) + BD
```

### 2.5.3 复杂度分析

T(n) = 3·T(n/2) + O(n)

由 Master 定理：n^(log₂ 3) ≈ n^1.585，f(n) = O(n) = O(n^(log₂ 3 - 0.585))

∴ **T(n) = Θ(n^(log₂ 3)) ≈ Θ(n^1.585)**，优于传统的 Θ(n²)。

## 2.6 Strassen 矩阵乘法

### 2.6.1 问题描述

两个 n×n 矩阵相乘，标准算法 O(n³)。

### 2.6.2 分治策略

将每个 n×n 矩阵分成 4 个 n/2 × n/2 的子矩阵：

```
┌     ┐   ┌     ┐ ┌     ┐
│C₁₁ C₁₂│   │A₁₁ A₁₂│ │B₁₁ B₁₂│
│C₂₁ C₂₂│ = │A₂₁ A₂₂│×│B₂₁ B₂₂│
└     ┘   └     ┘ └     ┘
```

标准分治需要 8 次 n/2 矩阵乘法。

**Strassen 算法**：通过巧妙的 7 次乘法组合，将 8 次减为 7 次：

```
P₁ = A₁₁ × (B₁₂ - B₂₂)
P₂ = (A₁₁ + A₁₂) × B₂₂
P₃ = (A₂₁ + A₂₂) × B₁₁
P₄ = A₂₂ × (B₂₁ - B₁₁)
P₅ = (A₁₁ + A₂₂) × (B₁₁ + B₂₂)
P₆ = (A₁₂ - A₂₂) × (B₂₁ + B₂₂)
P₇ = (A₁₁ - A₂₁) × (B₁₁ + B₁₂)

C₁₁ = P₅ + P₄ - P₂ + P₆
C₁₂ = P₁ + P₂
C₂₁ = P₃ + P₄
C₂₂ = P₅ + P₁ - P₃ - P₇
```

### 2.6.3 复杂度分析

T(n) = 7·T(n/2) + O(n²)

n^(log₂ 7) ≈ n^2.807，f(n) = O(n²) = O(n^(log₂ 7 - 0.807))

∴ **T(n) = Θ(n^(log₂ 7)) ≈ Θ(n^2.807)**，优于传统的 Θ(n³)。

## 2.7 棋盘覆盖问题

### 2.7.1 问题描述

在一个 2ᵏ × 2ᵏ 的棋盘中，恰有一个方格是特殊方格（缺失）。要求用 L 形骨牌覆盖其余所有方格。

L 形骨牌由 3 个方格组成，有 4 种旋转形态。

### 2.7.2 分治策略

1. 将 2ᵏ × 2ᵏ 棋盘分成 4 个 2ᵏ⁻¹ × 2ᵏ⁻¹ 子棋盘
2. 特殊方格必在其中一个子棋盘中
3. 在棋盘中心放一个 L 形骨牌，使其覆盖另外 3 个子棋盘各一个方格（将它们变成有特殊方格的子棋盘）
4. 递归处理 4 个子棋盘

### 2.7.3 复杂度分析

T(k) = 4T(k-1) + O(1)，展开得 T(k) = (4ᵏ - 1)/3 = Θ(4ᵏ)

由于棋盘格子总数 n = 4ᵏ，所以 **T(n) = Θ(n)**，线性时间！

## 2.8 本章习题

### 习题1

> 用分治法求数组 a[0..n-1] 中的最大值和最小值，使比较次数尽可能少。

**解**：将数组分成两半，分别求出左右两半的最大值和最小值，然后合并。

```c
void MaxMin(int a[], int l, int r, int *max, int *min) {
    if (l == r) {
        *max = *min = a[l];
    } else if (l + 1 == r) {
        if (a[l] > a[r]) { *max = a[l]; *min = a[r]; }
        else { *max = a[r]; *min = a[l]; }
    } else {
        int mid = (l + r) / 2;
        int lmax, lmin, rmax, rmin;
        MaxMin(a, l, mid, &lmax, &lmin);
        MaxMin(a, mid + 1, r, &rmax, &rmin);
        *max = (lmax > rmax) ? lmax : rmax;
        *min = (lmin < rmin) ? lmin : rmin;
    }
}
```

比较次数：T(n) = 2T(n/2) + 2，解得 T(n) = 3n/2 - 2，优于直接遍历的 2n-2 次比较。

### 习题2

> 设计一个 O(log n) 算法，在旋转有序数组（如 [4,5,6,7,0,1,2]）中查找目标值。

**解**：每次二分的 mid 必然将数组分成一个有序段和一个旋转段。

```c
int SearchRotated(int a[], int n, int target) {
    int l = 0, r = n - 1;
    while (l <= r) {
        int mid = (l + r) / 2;
        if (a[mid] == target) return mid;
        
        if (a[l] <= a[mid]) {  // 左半有序
            if (a[l] <= target && target < a[mid]) r = mid - 1;
            else l = mid + 1;
        } else {  // 右半有序
            if (a[mid] < target && target <= a[r]) l = mid + 1;
            else r = mid - 1;
        }
    }
    return -1;
}
```

时间复杂度 **O(log n)**。

### 习题3

> 写出快速排序在最好情况下的递归方程，并求解。

**解**：最好情况下每次划分恰好将序列对半分。

T(n) = 2T(n/2) + O(n)

Master定理：a=2, b=2, n^(log₂ 2)=n = f(n)=Θ(n)，情况2

∴ **T(n) = Θ(n log n)**

### 习题4

> 用分治法计算 xⁿ 需要多少次乘法？

**解**：

```
xⁿ = 
  x^(n/2) × x^(n/2)       当 n 为偶数
  x^((n-1)/2) × x^((n-1)/2) × x  当 n 为奇数
```

每次递归将问题规模减半：T(n) = T(n/2) + O(1)

解得 **T(n) = O(log n)** 次乘法，远优于直接乘的 O(n)。
