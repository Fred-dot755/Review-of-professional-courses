# 第4章 指令系统

> 指令系统（Instruction Set Architecture，ISA）是计算机硬件与软件之间的接口。它定义了处理器能执行的所有指令的集合。理解指令系统是理解CPU如何工作的前提，也是软硬件协同设计的关键。

---

## 4.1 指令系统概述

### 4.1.1 什么是指令系统

指令系统是一台计算机能执行的全部指令的集合，它规定了：

- 指令的**格式**（指令编码方式）
- 指令的**操作类型**（能做什么运算）
- 指令的**寻址方式**（如何找到操作数）
- 可访问的**寄存器集合**
- **数据类型**和**数据宽度**

### 4.1.2 指令系统的发展方向

| 类型 | 全称 | 特点 | 代表 |
|------|------|------|------|
| **CISC** | Complex Instruction Set Computer | 指令数量多、功能复杂、长度可变 | x86 (Intel/AMD) |
| **RISC** | Reduced Instruction Set Computer | 指令数量少、功能简单、定长编码 | ARM、MIPS、RISC-V |

| 对比维度 | CISC | RISC |
|----------|------|------|
| 指令数量 | 多（数百条） | 少（几十~一百多条） |
| 指令长度 | 可变 | 定长（通常32位） |
| 寻址方式 | 多种 | 少 |
| 指令执行周期 | 有的指令需多周期 | 大多数单周期 |
| 编译优化 | 复杂 | 更依赖编译器 |
| 设计哲学 | 硬件做更多 | 软件（编译器）做更多 |

> **现实**：现代处理器早已模糊了CISC和RISC的界限。Intel CPU内部将CISC指令译码为类似RISC的微操作（$\mu$OP），而ARM也在不断增加复杂指令。真正的趋势是取两者之长。

## 4.2 指令格式

### 4.2.1 指令的一般格式

一条机器指令的一般格式：

```
┌──────────────┬──────────────────┐
│   操作码 OP   │    地址码 A       │
└──────────────┴──────────────────┘
```

- **操作码（OP, Operation Code）**：指明指令做什么操作（加减乘除、传送、转移等）
- **地址码（A, Address Code）**：指明操作数的地址或操作数本身

### 4.2.2 按地址数分类

| 类型 | 格式 | 特点 |
|------|------|------|
| **三地址指令** | `OP A1, A2, A3` | `A1 ← (A2) OP (A3)` |
| **二地址指令** | `OP A1, A2` | `A1 ← (A1) OP (A2)`，结果覆盖一个操作数 |
| **一地址指令** | `OP A1` | `ACC ← (ACC) OP (A1)`，隐式使用累加器 |
| **零地址指令** | `OP` | 无地址码，操作数隐含（如堆栈操作） |

```
示例（三地址）：ADD R1, R2, R3   →  R1 = R2 + R3
示例（二地址）：ADD R1, R2       →  R1 = R1 + R2
示例（一地址）：ADD R1           →  ACC = ACC + R1
示例（零地址）：ADD              →  栈顶两元素相加，结果压栈
```

### 4.2.3 操作码的编码

#### 定长操作码

操作码长度固定，位于指令的固定位置。译码简单，但编码空间利用率低。

#### 扩展操作码 ⭐

操作码长度可变，充分利用编码空间。

```
示例：16位指令，假设地址码各占6位

4位操作码：0000~1110 → 15条三地址指令（1111留给扩展用）
8位操作码：1111 000000~1111 111110 → 63条二地址指令
...
```

> **核心原则**：短操作码一定不能是长操作码的前缀（否则译码时会产生歧义）。

## 4.3 寻址方式 ⭐⭐⭐

寻址方式是确定本条指令操作数地址及下一条要执行指令地址的方法。

### 4.3.1 指令寻址

| 方式 | 原理 | 下一条指令地址 |
|------|------|---------------|
| **顺序寻址** | PC + "1"（"1"指一个指令字的长度） | `PC ← PC + 1` |
| **跳跃寻址** | 由跳转指令给出目标地址 | `PC ← 转移目标地址` |

### 4.3.2 数据寻址（操作数寻址）⭐

| 寻址方式 | 形式地址含义 | 有效地址EA | 用途 |
|----------|-------------|-----------|------|
| **立即寻址** | 操作数本身 | 无（直接使用） | 常量赋值 |
| **直接寻址** | 操作数的地址 | `EA = A` | 访问固定地址 |
| **间接寻址** | 存放操作数地址的存储单元地址 | `EA = (A)` | 指针访问 |
| **寄存器寻址** | 寄存器编号 | `EA = R_i`（操作数在寄存器中） | 最常用 |
| **寄存器间接寻址** | 寄存器编号 | `EA = (R_i)` | 数组遍历 |
| **变址寻址** | 变址值 | `EA = (IX) + A` | 数组元素访问 |
| **基址寻址** | 基址值 | `EA = (BR) + A` | 程序重定位 |
| **相对寻址** | 位移量 | `EA = (PC) + A` | 相对跳转、可重入代码 |
| **堆栈寻址** | 隐含 | `EA = (SP)` | 函数调用参数 |

#### 重点区分：变址寻址 vs 基址寻址

| 维度 | 变址寻址 | 基址寻址 |
|------|----------|----------|
| 加法含义 | 指令中地址A为基址，变址寄存器IX为偏移 | 基址寄存器BR为基址，指令中地址A为偏移 |
| 可变部分 | 变址寄存器（程序员可改） | 指令中的A（编译时确定） |
| 不变部分 | 指令中的A（编译时确定） | 基址寄存器（由OS管理） |
| 典型用途 | 数组遍历（循环中IX变化） | 程序浮动、多道程序 |

#### 相对寻址

$$EA = (PC) + A$$

其中 $A$ 是补码表示的位移量（可正可负）。**特别重要**：这里的 `(PC)` 是PC更新后的值（即取下一条指令的地址）。

于跳跃范围通常较小，可以缩短指令中的地址码长度，也便于程序的动态重定位。

### 4.3.3 寻址方式综合示例

以一条假设的 `ADD` 指令为例，说明不同寻址方式的效果（主存2000号单元存放操作数300）：

| 寻址方式 | 指令示例 | 操作过程 |
|----------|----------|----------|
| 立即寻址 | `ADD #300` | ACC ← ACC + 300 |
| 直接寻址 | `ADD 2000` | ACC ← ACC + [2000] |
| 间接寻址 | `ADD @2000` | ACC ← ACC + [[2000]] |
| 寄存器寻址 | `ADD R1` | ACC ← ACC + R1 |
| 寄存器间接 | `ADD (R1)` | ACC ← ACC + [R1] |
| 变址寻址 | `ADD 1000(IX)` | ACC ← ACC + [1000+IX] |

## 4.4 指令的分类

### 4.4.1 按操作功能分类

| 类型 | 示例操作 | 典型指令 |
|------|----------|----------|
| **数据传送** | 寄存器间、寄存器与内存间 | `MOV`, `LOAD`, `STORE`, `PUSH`, `POP` |
| **算术运算** | 加减乘除、比较 | `ADD`, `SUB`, `MUL`, `DIV`, `CMP` |
| **逻辑运算** | 与或非、移位 | `AND`, `OR`, `NOT`, `XOR`, `SHL`, `SHR` |
| **程序控制** | 跳转、调用、返回 | `JMP`, `JE`, `CALL`, `RET`, `INT` |
| **输入输出** | I/O端口读写 | `IN`, `OUT` |
| **系统控制** | 特权指令 | `HLT`, `CLI`, `STI` |

### 4.4.2 条件转移指令

条件转移通常基于**标志寄存器（FLAGS/PSW）**中的条件码：

| 标志位 | 名称 | 含义 |
|--------|------|------|
| **ZF**（Zero Flag） | 零标志 | 结果为0时置1 |
| **CF**（Carry Flag） | 进位/借位标志 | 无符号数运算进位/借位 |
| **OF**（Overflow Flag） | 溢出标志 | 有符号数运算溢出 |
| **SF**（Sign Flag） | 符号标志 | 结果的最高位 |
| **PF**（Parity Flag） | 奇偶标志 | 结果中1的个数为偶数时置1 |

常见条件转移指令：

| x86指令 | 条件 | 含义 |
|---------|------|------|
| `JE` / `JZ` | ZF=1 | 等于/为零时转移 |
| `JNE` / `JNZ` | ZF=0 | 不等/非零时转移 |
| `JG` | ZF=0 且 SF=OF | 有符号大于时转移 |
| `JL` | SF≠OF | 有符号小于时转移 |
| `JA` | CF=0 且 ZF=0 | 无符号大于时转移 |
| `JB` | CF=1 | 无符号小于时转移 |

### 4.4.3 子程序调用指令

| 指令 | 操作 |
|------|------|
| `CALL addr` | `PUSH` 返回地址（下一指令PC值）；`PC ← addr` |
| `RET` | `POP PC`（从栈中弹出返回地址） |

> **调用与跳转的本质区别**：`CALL` 需要**保存返回地址**（以便 `RET` 恢复），而 `JMP` 不需要。

## 4.5 典型指令系统简介

### 4.5.1 x86/x86-64 指令系统

- **类型**：CISC
- **特点**：指令长度可变（1~15字节），支持复杂的寻址方式，兼容历史悠久
- **寄存器**：x86-64有16个64位通用寄存器（RAX, RBX, RCX, RDX, RSI, RDI, RBP, RSP, R8~R15）

### 4.5.2 ARM 指令系统

- **类型**：RISC
- **特点**：定长32位指令（Thumb模式下16位），Load/Store架构，条件执行
- **应用**：移动设备、嵌入式系统、Apple Silicon（M1/M2/M3系列）

### 4.5.3 RISC-V 指令系统

- **类型**：RISC
- **特点**：开源ISA，模块化设计，基础整数指令集（RV32I/RV64I）+ 标准扩展
- **扩展**：M（乘除法）、F/D（单/双精度浮点）、A（原子操作）、C（压缩指令）等

---

## 4.6 RISC vs CISC：深度对比

### 4.6.1 RISC的主要特征

1. **Load/Store架构**：只有LOAD和STORE指令访问内存，其他指令在寄存器间操作
2. **定长指令**：便于流水线设计
3. **大量通用寄存器**：通常32个以上
4. **简单的寻址方式**：减少指令复杂度
5. **延迟转移/加载**：将转移/加载的延迟暴露给编译器优化

### 4.6.2 CISC的主要特征

1. **指令可直接操作内存**：如 `ADD AX, [BX]`
2. **可变长指令**：提高代码密度
3. **复杂的寻址方式**：一条指令可完成复杂操作
4. **微程序控制**：复杂指令通过微程序实现

### 4.6.3 现代趋势

```
现代处理器融合了两种设计哲学：

    CISC指令（x86）
         │
         ▼
    ┌─────────────┐
    │ 硬件译码器   │ → 将CISC指令翻译为微操作（μOP）
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ RISC-like   │
    │ 执行核心     │
    │ (μOPs)      │
    └─────────────┘
```

## 本章小结

指令系统是软硬件之间的"契约"，本章核心要点：

| 知识点 | 要点 |
|--------|------|
| **指令格式** | 操作码 + 地址码；定长 vs 扩展操作码 |
| **寻址方式** | 10种常见寻址方式，重点区分变址寻址/基址寻址/相对寻址 |
| **指令分类** | 数据传送、算术逻辑、程序控制、I/O、系统控制 |
| **CISC vs RISC** | 两大设计哲学各有优势，现代CPU趋向融合 |
| **关键理解** | `CALL` vs `JMP` 的区别在于是否保存返回地址 |
