# 第5章 回溯法

## 5.1 回溯法基本思想

### 5.1.1 什么是回溯法

**回溯法**（Backtracking）是一种系统地搜索问题解空间的算法。它采用**深度优先**策略，从根节点出发搜索解空间树。当搜索到某一节点时，先判断该节点是否可能包含问题的解（剪枝），如果肯定不包含，则跳过对该子树（即以该节点为根的子树）的搜索，逐层向其祖先节点**回溯**；否则进入该子树，继续按深度优先策略搜索。

```
回溯法 = 深度优先搜索 + 剪枝函数
```

### 5.1.2 解空间与解空间树

- **解空间**：问题所有可能解构成的集合
- **解空间树**：将解空间组织成树形结构

常见的解空间树有两种：

| 类型 | 结构 | 典型问题 |
|------|------|----------|
| **子集树** | 每个节点选择或不选择某个元素（二叉树） | 0-1背包、子集和 |
| **排列树** | 每个节点选择一个排列位置（多叉树） | 旅行商、n皇后、批处理调度 |

### 5.1.3 剪枝函数

剪枝函数用于减少搜索空间，分为两类：

- **约束函数**（Constraint Function）：排除不满足约束条件的子树
- **限界函数**（Bound Function）：排除不可能产生最优解的子树

### 5.1.4 回溯法的一般框架

#### 子集树框架

```c
void Backtrack(int t) {  // t为当前层数
    if (t > n) {
        // 到达叶节点，更新最优解
        UpdateBest();
    } else {
        for (int i = 0; i <= 1; i++) {  // 0:不选, 1:选
            x[t] = i;
            if (Constraint(t) && Bound(t))
                Backtrack(t + 1);
        }
    }
}
```

#### 排列树框架

```c
void Backtrack(int t) {
    if (t > n) {
        UpdateBest();
    } else {
        for (int i = t; i <= n; i++) {
            Swap(x[t], x[i]);
            if (Constraint(t) && Bound(t))
                Backtrack(t + 1);
            Swap(x[t], x[i]);  // 回溯
        }
    }
}
```

### 5.1.5 复杂度分析

- 子集树叶子数：最多 2ⁿ
- 排列树叶子数：最多 n!

**回溯法在最坏情况下是指数级复杂度**，但通过剪枝可大幅减少搜索量。

## 5.2 n 皇后问题

### 5.2.1 问题描述

在 n×n 的棋盘上放置 n 个皇后，使得任意两个皇后不能同行、同列、同对角线。求所有合法放置方案。

### 5.2.2 解空间结构

用一维数组 x[1..n] 表示解，x[i] = j 表示第 i 行皇后放在第 j 列。解空间为**排列树**（n! 个叶子）。

### 5.2.3 约束条件

两个皇后 (i, x[i]) 和 (j, x[j]) 冲突的条件：

- **同列**：x[i] = x[j]
- **同对角线**：|i - j| = |x[i] - x[j]|

### 5.2.4 算法实现

```c
int x[N];  // x[i]表示第i行皇后所在的列
int count = 0;  // 解的总数

// 判断第k个皇后与前面k-1个是否冲突
int Place(int k) {
    for (int i = 1; i < k; i++) {
        if (x[i] == x[k] || abs(i - k) == abs(x[i] - x[k]))
            return 0;
    }
    return 1;
}

void NQueens(int k, int n) {
    if (k > n) {
        count++;
        // 输出解
        for (int i = 1; i <= n; i++)
            printf("%d ", x[i]);
        printf("\n");
    } else {
        for (int j = 1; j <= n; j++) {
            x[k] = j;
            if (Place(k))
                NQueens(k + 1, n);
        }
    }
}
```

### 5.2.5 经典结果

| n | 解的总数 |
|---|----------|
| 1 | 1 |
| 4 | 2 |
| 5 | 10 |
| 6 | 4 |
| 7 | 40 |
| 8 | 92 |
| 9 | 352 |
| 10 | 724 |

## 5.3 0-1 背包问题（回溯法）

### 5.3.1 问题描述

同第3章 0-1背包问题。用回溯法在解空间树（子集树）中搜索最优解。

### 5.3.2 剪枝策略

- **约束函数**：当前重量 + w[t] ≤ 背包容量 C
- **限界函数**：当前价值 + 剩余物品的**上界估计值** > 当前最优值（否则剪枝）

上界估计值：将剩余物品按单位价值排序，尽量装入（分数背包贪心）。

### 5.3.3 算法实现

```c
int n, C;          // 物品数, 容量
int w[N], v[N];    // 重量, 价值
int bestv = 0;     // 当前最优价值
int cw = 0, cv = 0;  // 当前重量, 当前价值

// 计算上界
double Bound(int i) {
    double cleft = C - cw;  // 剩余容量
    double bound = cv;      // 当前价值
    
    // 按单位价值降序遍历剩余物品
    while (i <= n && w[i] <= cleft) {
        cleft -= w[i];
        bound += v[i];
        i++;
    }
    // 装满剩余容量（分数部分）
    if (i <= n)
        bound += v[i] * cleft / w[i];
    return bound;
}

void KnapsackBT(int i) {
    if (i > n) {
        bestv = cv;  // 更新最优值
        return;
    }
    
    // 搜索左子树（装物品i）
    if (cw + w[i] <= C) {  // 约束函数
        cw += w[i];
        cv += v[i];
        KnapsackBT(i + 1);
        cw -= w[i];  // 回溯
        cv -= v[i];
    }
    
    // 搜索右子树（不装物品i）
    if (Bound(i + 1) > bestv)  // 限界函数
        KnapsackBT(i + 1);
}
```

### 5.3.4 复杂度分析

最坏复杂度 O(2ⁿ)，但实际通过剪枝远小于此。

## 5.4 图的 m 着色问题

### 5.4.1 问题描述

给定无向连通图 G 和 m 种颜色，找出所有使相邻顶点不同色的着色方案。

### 5.4.2 算法实现

```c
int n, m;              // 顶点数, 颜色数
int graph[N][N];       // 邻接矩阵
int color[N];          // color[i] 表示顶点i的颜色(1..m)
int count = 0;         // 解的数量

// 检查顶点t着色color[t]后是否与相邻顶点冲突
int OK(int t) {
    for (int j = 1; j <= n; j++) {
        if (graph[t][j] && color[t] == color[j])
            return 0;
    }
    return 1;
}

void MColoring(int t) {
    if (t > n) {
        count++;
        // 输出着色方案
        for (int i = 1; i <= n; i++)
            printf("%d ", color[i]);
        printf("\n");
    } else {
        for (int i = 1; i <= m; i++) {
            color[t] = i;
            if (OK(t))
                MColoring(t + 1);
        }
    }
}
```

### 5.4.3 复杂度分析

搜索树的节点数最多为 mⁿ，通过剪枝可大幅减少。

## 5.5 旅行商问题（TSP）— 回溯法

### 5.5.1 问题描述

给定 n 个城市及两两之间的距离（完全图），求从城市 1 出发，经过每个城市恰好一次并回到城市 1 的最短路径。

### 5.5.2 解空间结构

排列树。以城市 1 为起点，对剩余 n-1 个城市进行全排列。

### 5.5.3 剪枝策略

- **约束函数**：两个城市之间有边（连通）
- **限界函数**：当前路径长度 + 剩余城市到起点的最小可能距离 ≥ 当前最优值 → 剪枝

### 5.5.4 算法实现

```c
int n;
int graph[N][N];     // 距离矩阵
int x[N];            // 当前路径 x[1]=1 固定
int bestx[N];        // 最优路径
int bestc = INF;     // 最优代价
int cc = 0;          // 当前代价

void TSP_Backtrack(int t) {
    if (t == n) {  // 已排列完n-1个城市(加上1共n个)
        // 需要构成回路：最后一个城市到城市1有边，且总代价更优
        if (graph[x[n-1]][x[n]] != INF && graph[x[n]][1] != INF &&
            cc + graph[x[n-1]][x[n]] + graph[x[n]][1] < bestc) {
            bestc = cc + graph[x[n-1]][x[n]] + graph[x[n]][1];
            for (int j = 1; j <= n; j++)
                bestx[j] = x[j];
        }
    } else {
        for (int j = t; j <= n; j++) {
            // 约束 + 限界
            if (graph[x[t-1]][x[j]] != INF &&
                cc + graph[x[t-1]][x[j]] < bestc) {
                Swap(x[t], x[j]);
                cc += graph[x[t-1]][x[t]];
                TSP_Backtrack(t + 1);
                cc -= graph[x[t-1]][x[t]];
                Swap(x[t], x[j]);  // 回溯
            }
        }
    }
}
```

### 5.5.5 复杂度

最坏 O(n!)，实际通过剪枝可求解中等规模问题（n≤20）。

## 5.6 子集和问题

### 5.6.1 问题描述

给定 n 个正整数 w₁, ..., wₙ 和一个目标值 M，找出所有和恰好为 M 的子集。

### 5.6.2 算法实现

```c
int n, M;
int w[N], x[N];  // x[i]=1表示选中
int sum = 0;     // 当前和

void SubsetSum(int t) {
    if (t > n) {
        if (sum == M) {
            // 输出解
            for (int i = 1; i <= n; i++)
                if (x[i]) printf("%d ", w[i]);
            printf("\n");
        }
    } else {
        // 选 w[t]
        if (sum + w[t] <= M) {  // 约束
            sum += w[t]; x[t] = 1;
            SubsetSum(t + 1);
            sum -= w[t];  // 回溯
        }
        // 不选 w[t]
        x[t] = 0;
        SubsetSum(t + 1);
    }
}
```

## 5.7 本章习题

### 习题1

> 对于 4 皇后问题，请画出回溯搜索过程中的解空间树（含剪枝标记）。

**解**：4 皇后问题共有 2 个解：(2,4,1,3) 和 (3,1,4,2)。

搜索过程（括号内为 x[1..k] 的当前赋值）：

```
Level 1: x[1]=1 → x[1]=2 → x[1]=3 → x[1]=4
  x[1]=1:
    x[2]=3: x[3]=✗(冲突)
    x[2]=4: x[3]=2: x[4]=✗(冲突)
  x[1]=2:
    x[2]=4:
      x[3]=1: x[4]=3 ✓ 解1: (2,4,1,3)
      x[3]=3: x[4]=✗
  x[1]=3:
    x[2]=1:
      x[3]=4: x[4]=2 ✓ 解2: (3,1,4,2)
  x[1]=4: (对称于x[1]=1, 无解)
```

### 习题2

> 用回溯法求解 0-1 背包：n=4, C=7, w=(3,5,2,1), v=(9,10,7,4)。画出搜索树，标出剪枝。

**解**：先按 v/w 排序：
- 物品1: w=2, v=7 (v/w=3.5)
- 物品2: w=3, v=9 (v/w=3.0)
- 物品3: w=1, v=4 (v/w=4.0)
- 物品4: w=5, v=10 (v/w=2.0)

排序后：
- 物品1: w=1, v=4 (v/w=4.0)
- 物品2: w=2, v=7 (v/w=3.5)
- 物品3: w=3, v=9 (v/w=3.0)
- 物品4: w=5, v=10 (v/w=2.0)

最优值：**bestv = 20**（装物品1+2+4: w=1+2+5=8>7 不行；最优装 1+3+4=1+3+5=9>7 不行；最优装 1+2+3=1+2+3=6≤7, v=4+7+9=**20**；或 2+4=2+5=7, v=7+10=**17**）

最优解：物品1(1,4), 物品2(2,7), 物品3(3,9)，总重=6, 总价值=20 ✓

### 习题3

> 回溯法求解 TSP 时，如何设计更紧的限界函数？

**解**：可以使用最小出边和法（或最小生成树下界法）。

对于每个未访问城市，取它到任意其他未访问城市（包括起点）的最短边的长度，求和后除以2（因为每条边被计算两次）。这个下界加上当前路径长度，若 ≥ 当前最优值则剪枝。

另一种方法：当前路径+剩余城市的最小生成树边权和+到起点最近边。这给出一个有效下界。
