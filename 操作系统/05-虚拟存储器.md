# 第5章：虚拟存储器

> 虚拟存储器（Virtual Memory）是现代操作系统最重要的技术之一，它使程序可以拥有比物理内存大得多的地址空间，实现了逻辑上的内存扩充。

---

## 5.1 虚拟存储器的基本概念

### 5.1.1 定义与特征

> **虚拟存储器**：具有请求调入功能和置换功能，能从逻辑上对内存容量加以扩充的一种存储器系统。

```
虚拟地址空间 vs 物理内存：

进程A的虚拟空间（4GB）    物理内存（512MB ~ 16GB）
┌──────────────────┐     ┌───────────────────┐
│  代码段 (2MB)     │ ──→ │  代码段 (已装入)     │
│  数据段 (1MB)     │ ──→ │  数据段 (已装入)     │
│  堆 (大量未用)     │     │  空闲               │
│  ...              │     │  ...                │
│  栈               │ ──→ │  栈 (已装入)        │
│  共享库 (128MB)   │ ──→ │  共享库 (部分装入)    │
└──────────────────┘     └───────────────────┘

按需装入，用不到的页不必加载到物理内存
```

### 5.1.2 虚拟存储器的三大特征 ⭐

| 特征 | 含义 |
|------|------|
| **多次性** | 一个程序可分多次调入内存 |
| **对换性** | 程序运行中可在内存与外存间换入换出 |
| **虚拟性** | 逻辑上扩充内存容量，虚拟空间可以远大于物理内存 |

> ⚠️ **虚拟存储器的容量** = `min(CPU地址总线宽度, 外存容量)`，而非无限大。

### 5.1.3 实现基础

虚拟存储器建立在**离散分配**的存储管理方式基础上：

| 实现方式 | 说明 |
|----------|------|
| **请求分页系统** | 在基本分页基础上增加请求调页和页面置换功能 |
| **请求分段系统** | 在基本分段基础上增加请求调段和分段置换功能 |
| **请求段页式系统** | 兼具两者 |

## 5.2 请求分页存储管理 ⭐⭐⭐

### 5.2.1 请求分页的硬件支持

| 硬件部件 | 功能 |
|----------|------|
| **页表** | 记录页的状态（是否在内存、是否被修改等） |
| **缺页中断机构** | 访问的页不在内存时产生中断 |
| **地址变换机构** | 逻辑地址 → 物理地址，含缺页检测 |

### 5.2.2 请求分页页表项 ⭐

```
增强的页表项：
┌──────┬────┬───┬───┬───┬───────────┐
│页框号 │存在位│访问位│修改位│保护位│外存地址    │
└──────┴────┴───┴───┴───┴───────────┘

存在位(P)：1=在内存，0=在外存 → 缺页
访问位(A)：用于页面置换算法（如Clock算法）
修改位(M)：1=被修改过，换出时需写回磁盘；0=未修改，可直接丢弃
保护位：读/写/执行权限
外存地址：该页在外存（交换区）中的位置
```

### 5.2.3 缺页中断 ⭐

> **缺页中断（Page Fault）**：当CPU访问的页不在内存中（存在位P=0）时产生的中断。

```
缺页中断处理流程：

CPU访问某页
     │
     ▼
查页表：该页在内存？ ──是──→ 继续执行
     │
    否（缺页中断）
     │
     ▼
① OS检查访问合法性 ──不合法──→ 终止进程（段错误）
     │
    合法
     │
     ▼
② 找一空闲页框
     │
     ├── 有空闲页框 → 直接分配
     │
     └── 无空闲 → 选一页置换
                   │
                   ├── 被选页未修改(M=0) → 直接覆盖
                   │
                   └── 被选页已修改(M=1) → 写回磁盘
     │
     ▼
③ 从磁盘读入所需页到页框
     │
     ▼
④ 更新页表（P=1, 更新页框号）
     │
     ▼
⑤ 重新执行引发缺页的指令（关键！）
```

> ⚠️ **缺页中断的特殊性**：
> - 在指令执行**过程中**产生（而非执行后）
> - 处理完成后须**重新执行**引发缺页的那条指令

### 5.2.4 地址变换流程（请求分页）

```
逻辑地址 A
    │
    ├── ① 越界检查（A < 程序长度？）
    │
    ├── ② 分拆：页号P + 页内偏移d
    │
    ├── ③ 查页表：页号P → 页表项
    │
    ├── ④ P位=0？ → 产生缺页中断 → OS处理（见上流程）
    │
    └── ⑤ P位=1 → 物理地址 = 页框号f × 页大小 + d → 访问内存
```

## 5.3 页面置换算法 ⭐⭐⭐

> 当内存空间不足需要换出页面时，选哪个页面换出最合理？

### 5.3.1 最佳置换算法（OPT）⭐

> **原则**：淘汰未来**最长时间内不再访问**的页面。

| 特点 | 说明 |
|------|------|
| **性质** | 理论最优，缺页率最低 |
| **可实现吗** | **不可实现**（无法预知未来访问序列） |
| **用途** | 作为衡量其他算法的**基准** |

### 5.3.2 先进先出置换算法（FIFO）

> **原则**：淘汰**最先进入**内存的页面。

| 特点 | 说明 |
|------|------|
| **优点** | 实现简单（用队列） |
| **缺点** | 可能淘汰频繁访问的页；可能产生**Belady异常** |

#### ⭐ Belady异常

> **Belady异常**：增加物理块数（内存页框）后，缺页次数反而增加。

```
页面引用串：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

3个页框（FIFO）：缺页9次
4个页框（FIFO）：缺页10次  ← 反而更多！
```

> ⚠️ **FIFO**可能产生Belady异常；**LRU、OPT**不会产生。

### 5.3.3 最近最久未使用（LRU）⭐⭐⭐

> **原则**：淘汰**最近最长时间未被访问**的页面。

```
实现方式：

① 寄存器法（移位寄存器）
   每次访存：右移一位，被访问页最高位置1
   值最小的页 = 最久未访问

② 栈法
   访问某页 → 将其移到栈顶
   栈底 = 最久未访问的页

③ 近似实现：Clock算法
```

| 特点 | 说明 |
|------|------|
| **优点** | 性能接近OPT，无Belady异常 |
| **缺点** | 硬件支持开销大（每次访存都需更新） |

### 5.3.4 Clock置换算法（NRU）⭐⭐

> LRU的**近似**实现，实现成本低，实用性强。

```
简单Clock（NRU）算法：

将内存中的页组织成循环链表
每页有访问位(A)
指针扫描：

    指针从当前位置扫描
        │
        ▼
    ┌──── A=0？ ──── 是 → 选中淘汰
    │
    │ A=1
    │   → 置 A=0
    │   → 指针下移
    │
    └── 重复扫描
```

#### 改进型Clock算法 ⭐

> 增加修改位(M)，两轮扫描，优先淘汰未修改的页。

```
优先级（由低到高）：
  第1类：A=0, M=0   ← 最优淘汰对象（未访问+未修改）
  第2类：A=0, M=1   ← 次优（未访问但被修改，需写回）
  第3类：A=1, M=0   ← 可能再访问，未修改
  第4类：A=1, M=1   ← 最差淘汰对象（近期访问过+被修改）

扫描过程：
  第一轮：找 A=0, M=0 → 找到淘汰，未找到则继续
  第二轮：找 A=0, M=1 → 扫描中将经过的A都置为0
  ... 若两轮没找到（不太可能），重新开始
```

### 5.3.5 其他置换算法

| 算法 | 原则 | 特点 |
|------|------|------|
| **LFU** | 淘汰访问次数最少的页 | 需计数器，考虑频率 |
| **MFU** | 淘汰访问次数最多的页 | 认为访问多的页已"完成使命" |
| **页面缓冲算法（PBA）** | 维护空闲链表和修改链表 | 减少I/O，实用（Solaris、VMS） |

### 5.3.6 页面置换算法总结

| 算法 | 原理 | Belady异常 | 实现难度 | 性能 |
|------|------|-----------|----------|------|
| **OPT** | 淘汰未来最远不用的 | 不会 | 不可实现 | 最优（理论） |
| **FIFO** | 淘汰最先进入的 | **会** | 极简单 | 差 |
| **LRU** | 淘汰最久未用的 | 不会 | 难 | 很好 |
| **Clock** | 近似LRU，用A位 | 不会 | 简单 | 好 |
| **改进Clock** | 考虑A+M位 | 不会 | 中等 | 更好 |
| **LFU** | 淘汰最少用的 | — | 中等 | 一般 |

## 5.4 工作集与驻留集

### 5.4.1 工作集（Working Set）

> **工作集**：进程在**最近 Δ 次**内存访问中所引用过的页面集合。

```
W(t, Δ) = { 在 (t-Δ, t] 时间内被访问的页 }

Δ 太小 → 工作集不全，频繁缺页
Δ 太大 → 工作集太大，浪费内存
```

| 概念 | 说明 |
|------|------|
| **核心思想** | 只需将工作集保持在内存中，即可保证进程高效运行 |
| **用途** | 指导进程应分配多少内存页框 |

### 5.4.2 驻留集

> **驻留集**：进程当前在内存中的页面集合。

| 分配策略 | 说明 |
|----------|------|
| **固定分配** | 进程的页框数不变（简单但不够灵活） |
| **可变分配** | 根据工作集动态调整页框数（灵活但复杂） |

### 5.4.3 抖动（Thrashing）⭐⭐

> **抖动**：进程频繁地换入换出页面，导致CPU大量时间浪费在页面置换上，而非执行程序。

```
抖动产生的原因：
  分配的页框数 < 工作集大小
  → 频繁缺页
  → CPU利用率下降
  → OS误以为CPU不够用，引入更多进程
  → 更少的内存分给每个进程
  → 缺页更频繁
  → ... 恶性循环
```

**解决抖动的策略**：
| 策略 | 说明 |
|------|------|
| **工作集策略** | 确保每个进程获得足够的页框来容纳工作集 |
| **缺页频率策略（PFF）** | 监控缺页率，过高则增加页框，过低则回收 |
| **挂起进程** | 将部分进程挂起到外存，释放内存 |

## 5.5 请求分段与段页式虚拟存储

### 5.5.1 请求分段系统

> 在基本分段的基础上增加请求调段和置换功能。

```
请求分段的硬件支持：
  - 段表项增加：存在位、修改位、访问位、增补位、外存地址
  - 缺段中断：类似缺页中断
  - 地址变换：类似请求分页
```

### 5.5.2 请求段页式系统

> 分页与分段的虚拟存储结合。

```
虚拟地址：│ 段号 S │ 页号 P │ 页内偏移 d │

优点：兼具分段（逻辑清晰、易共享保护）和分页（无外部碎片、虚拟存储）的优点
缺点：三次访存（段表→页表→数据），需用TLB加速
```

## 5.6 TLB与快表 ⭐⭐

### 5.6.1 TLB的基本概念

> **TLB（Translation Lookaside Buffer，快表）**：存放最近使用的页表项的CPU内部高速缓存。

```
地址变换（有TLB）：

CPU发出逻辑地址
       │
       ▼
  ┌─────────┐
  │  TLB    │  ← 先查TLB（命中率高，>99%）
  │ (快表)   │
  └────┬────┘
       │
  命中？──是──→ 直接得物理页框号 → 物理地址
       │
      否（TLB缺失）
       │
       ▼
  查页表（慢，在内存中）
       │ 可得物理页框号
       ▼
  更新TLB + 形成物理地址
```

| 项目 | 页表（慢表） | TLB（快表） |
|------|-------------|-------------|
| **位置** | 内存 | CPU内部（SRAM） |
| **容量** | 较大 | 很小（几十~几百项） |
| **速度** | 慢（一次访存） | 极快（ns级） |
| **内容** | 全部映射 | 最近使用的页表项缓存 |

### 5.6.2 TLB + Cache 联合访问

```
CPU ──逻辑地址──→ TLB ──物理地址──→ Cache ──→ 内存

组合情况：
 ① TLB命中 + Cache命中  → 最优（最快）
 ② TLB命中 + Cache缺失  → 需访存
 ③ TLB缺失 + Cache命中  → 也可能发生
 ④ TLB缺失 + Cache缺失  → 最差
```
