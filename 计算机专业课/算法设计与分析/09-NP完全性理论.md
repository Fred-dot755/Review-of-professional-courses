# 第9章 NP完全性理论

---

## 9.1 计算复杂性导论

### 9.1.1 为什么需要NP理论

算法设计与分析中，有些问题存在多项式时间算法（如排序 O(n log n)，最短路径 O(n²)），但有些问题至今未找到多项式算法（如旅行商、哈密顿回路）。**NP完全性理论**帮助判断一个问题是否"本质上很难"，避免徒劳地寻找高效算法。

### 9.1.2 问题分类

```
所有计算问题
├── 不可判定问题（Undecidable）
│   例：停机问题（Halting Problem）
│
└── 可判定问题（Decidable）
    ├── P 类问题（多项式时间可解）
    │   例：排序、最短路径、最大流
    │
    └── NP 类问题（多项式时间可验证）
        ├── P ⊆ NP
        ├── NP-完全（NPC）— NP中最难
        │   例：SAT、TSP、哈密顿回路
        └── NP-困难（NP-Hard）— 至少和NPC一样难
            例：停机问题（不可判定但NP-Hard）、TSP优化版
```

## 9.2 P 问题和 NP 问题

### 9.2.1 P 类（Polynomial Time）

> **定义**：P 类问题是指能在**确定性**图灵机上用**多项式时间**求解的问题。

换言之，存在一个算法，其时间复杂度为 O(nᵏ)（k 为常数），能在合理时间内解决该问题。

**典型 P 类问题**：
- 排序：O(n log n)
- 最短路径（Dijkstra）：O(n²)
- 最小生成树：O(E log V)
- 线性规划：O(n³)（椭球算法）
- 质数判定：O(log¹² n)（AKS算法）

### 9.2.2 NP 类（Nondeterministic Polynomial Time）

> **定义**：NP 类问题是指能在**非确定性**图灵机上用多项式时间求解的问题。

**等价定义**：给定一个解（证书/certificate），能在多项式时间内**验证**其正确性。

**典型 NP 问题**：
- 旅行商决策版：是否存在长度 ≤ K 的哈密顿回路？（验证：O(n) 检查路径）
- SAT（布尔可满足性）：给定布尔公式，是否存在一组赋值使其为真？
- 背包决策版：是否存在总价值 ≥ V 的装包方案？
- 图的 k-着色：能否用 k 种颜色着色？

### 9.2.3 P vs NP 问题

> **P = NP?** 这是计算机科学中最著名的未解决问题之一，也是 Clay 数学研究所的七大千禧年难题之一（悬赏 100 万美元）。

普遍认为 **P ≠ NP**，即有些问题验证容易但求解难。

## 9.3 多项式归约

### 9.3.1 什么是归约

> **定义**：问题 A 可以**多项式归约**到问题 B（记为 A ≤ₚ B），如果存在一个多项式时间的变换函数 f，使得对于 A 的任何实例 x：
>
> x 是 A 的 "是" 实例 ⇔ f(x) 是 B 的 "是" 实例

**直观理解**：如果 A ≤ₚ B，则 B 至少和 A 一样难。

- 若 B 可多项式求解，则 A 也可多项式求解
- 若 A 很难（无多项式解），则 B 也一定很难

### 9.3.2 归约的作用

```
如果问题 A ≤ₚ 问题 B，且：
  - B ∈ P  →  A ∈ P
  - B ∈ NP →  A ∈ NP
  - A 是 NP-完全 → B 是 NP-困难（若 B ∈ NP 则 B 也是 NP-完全）
```

---

## 9.4 NP-完全（NPC）问题

### 9.4.1 定义

> 问题 L 是 **NP-完全**（NP-Complete, NPC）的，当且仅当：
> 1. L ∈ NP（可在多项式时间内验证）
> 2. 对任意 L' ∈ NP，有 L' ≤ₚ L（NP-困难）

### 9.4.2 第一个 NPC 问题：SAT

**Cook-Levin定理**（1971）：布尔可满足性问题（SAT）是 NP-完全的。

Cook 证明了任何 NP 问题都可以多项式归约到 SAT。这是 NP 理论的基石。

### 9.4.3 证明一个问题是 NPC 的步骤

```
要证明问题 X 是 NP-完全的：
1. 证明 X ∈ NP（给出多项式时间验证算法）
2. 选择一个已知的 NPC 问题 Y
3. 证明 Y ≤ₚ X（构造多项式归约）
```

### 9.4.4 NPC 问题族谱

```
                  SAT（Cook 1971）
                   │
         ┌─────────┼─────────┐
         ↓         ↓         ↓
       3-SAT   3-Dimensional  哈密顿回路
                   Matching      │
         │                       ↓
    ┌────┼────┐               TSP
    ↓    ↓    ↓
  顶点   团   独立集
  覆盖  (Clique) (IS)
    │
    ↓
  子集和
    │
    ↓
  背包问题
    │
    ↓
  划分问题
    │
    ↓
  调度问题
```

## 9.5 经典 NPC 问题

### 9.5.1 SAT（布尔可满足性）

**问题**：给定一个合取范式（CNF）布尔公式，是否存在一组变量赋值使其为真？

例：(x₁ ∨ ¬x₂) ∧ (¬x₁ ∨ x₂ ∨ x₃) ∧ (¬x₃)

### 9.5.2 3-SAT

**问题**：每个子句恰好有 3 个文字的 SAT 问题。

**已知**：3-SAT 是 NPC 的（SAT ≤ₚ 3-SAT）。

### 9.5.3 团问题（Clique）

**问题**：给定无向图 G 和整数 k，G 中是否存在大小为 k 的团（完全子图）？

### 9.5.4 顶点覆盖（Vertex Cover）

**问题**：给定无向图 G 和整数 k，是否存在大小 ≤ k 的顶点集覆盖所有边？

**已知**：顶点覆盖是 NPC 的（3-SAT ≤ₚ 顶点覆盖）。

### 9.5.5 哈密顿回路（Hamiltonian Cycle）

**问题**：给定无向图 G，是否存在一条经过每个顶点恰好一次的回路？

**已知**：哈密顿回路是 NPC 的（3-SAT ≤ₚ 哈密顿回路）。

### 9.5.6 旅行商问题（TSP）决策版

**问题**：给定 n 个城市、距离矩阵和一个界限 K，是否存在总距离 ≤ K 的旅行路线？

**已知**：TSP 是 NPC 的（哈密顿回路 ≤ₚ TSP）。

### 9.5.7 子集和问题（Subset Sum）

**问题**：给定 n 个正整数和 S，是否存在和为 S 的子集？

**已知**：子集和是 NPC 的（3-SAT ≤ₚ 子集和）。

### 9.5.8 0-1 背包决策版

**问题**：给定物品重量 wᵢ、价值 vᵢ、容量 C 和目标价值 V，是否存在总价值 ≥ V 的装包方案？

**已知**：0-1 背包决策版是 NPC 的（子集和 ≤ₚ 0-1背包）。

## 9.6 NP-困难（NP-Hard）问题

### 9.6.1 定义

> 问题 L 是 **NP-困难** 的，如果所有 NP 问题都可以多项式归约到 L。L 不一定在 NP 中。

**NP-完全 = NP-困难 ∩ NP**

### 9.6.2 常见 NP-Hard 问题

- **TSP 优化版**：求最短哈密顿回路（不是 NP 问题，因为无法在多项式时间内验证"最优"）
- **停机问题**：不可判定但 NP-Hard
- **0-1背包优化版**：求最大价值（同样无法多项式验证最优）

## 9.7 P问题、NP问题、NPC问题关系图

```
┌─────────────────────────────────────────────┐
│                   NP-Hard                    │
│   ┌─────────────────────────────┐            │
│   │            NP                │           │
│   │   ┌───────────────┐         │           │
│   │   │     NPC        │         │           │
│   │   │  ┌─────┐      │         │           │
│   │   │  │  P  │      │         │           │
│   │   │  └─────┘      │         │           │
│   │   └───────────────┘         │           │
│   └─────────────────────────────┘            │
└─────────────────────────────────────────────┘

如果 P ≠ NP（普遍认为）:
  P ⊂ NPC ⊂ NP ⊂ NP-Hard

如果 P = NP（不太可能）:
  P = NP = NPC ⊆ NP-Hard
```

## 9.8 处理 NPC 问题的策略

既然 NPC 问题很可能没有多项式时间最优算法，实际中采用以下策略：

| 策略 | 说明 | 示例 |
|------|------|------|
| **近似算法** | 多项式时间求近似最优解，有性能保证 | TSP的Christofides算法（3/2近似） |
| **随机化算法** | 引入随机性，期望意义上高效 | 随机化快排、SAT的随机游走 |
| **参数化算法** | 将指数复杂度限制在某个小参数上 | 顶点覆盖的 O(1.27ᵏ + n) 算法 |
| **启发式算法** | 无理论保证但实践中有效 | 遗传算法、模拟退火、蚁群算法 |
| **限制输入** | 对特殊输入设计多项式算法 | 树上的NPC问题、区间图 |
| **分支限界/回溯** | 智能搜索 + 剪枝，小规模可精确求解 | 回溯法解TSP（n≤20） |

## 9.9 本章习题

### 习题1

> 解释为什么"验证一个解"和"找到一个解"可能是完全不同的难度。

**答**：以数独为例：

- **验证**：给定一个填好的数独，检查每一行、每一列、每一个宫是否有重复数字 → O(n²) 时间
- **求解**：找到一个有效填法 → 目前已知最好算法是指数级的（回溯法）

这就是 NP 问题的本质：解可以在多项式时间**验证**，但不一定能在多项式时间**找到**。

### 习题2

> 证明顶点覆盖问题（VC）的补问题是独立集问题（IS）。即：S 是图 G 的顶点覆盖 ⇔ V\S 是 G 的独立集。

**证明**：

(⇒) 设 S 是 G 的顶点覆盖。假设 V\S 不是独立集，则存在一条边 (u, v) 使得 u, v ∈ V\S，即 u, v ∉ S。但 S 覆盖了所有边，这意味着 (u, v) 中至少有一个端点在 S 中，矛盾。

(⇐) 设 V\S 是独立集。对任意边 (u, v)，由于 V\S 中没有两个端点相邻，所以 u 和 v 不能同时在 V\S 中。即至少有一个在 S 中，因此 S 覆盖了所有边。∎

### 习题3

> 将子集和问题归约到 0-1 背包决策版。

**解**：

给定子集和实例：n 个数 a₁,...,aₙ 和目标 S。

构造 0-1 背包实例：
- wᵢ = vᵢ = aᵢ（重量 = 价值）
- 容量 C = S
- 目标价值 V = S

则：存在子集和为 S ⇔ 存在 0-1 背包解使总价值 ≥ S（且能证明恰好 = S，因为 wᵢ = vᵢ 且 C = S）

该变换显然在多项式时间 O(n) 内完成。∎
