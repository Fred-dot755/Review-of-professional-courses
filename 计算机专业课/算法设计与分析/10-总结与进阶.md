## 10.1 五大算法设计策略全景对比

| 维度 | 分治法 | 动态规划 | 贪心算法 | 回溯法 | 分支限界法 |
|------|--------|----------|----------|--------|------------|
| **核心思想** | 分解独立子问题，合并解 | 解决重叠子问题，填表存储 | 局部最优 → 全局最优 | DFS搜索 + 剪枝 | BFS/最佳优先 + 限界 |
| **子问题关系** | 独立、不重叠 | 重叠、共享 | 贪心选择后只剩一个 | 解空间树中的节点 | 解空间树中的节点 |
| **求解方向** | 自顶向下递归 | 自底向上递推 | 自顶向下单步 | 自顶向下深度优先 | 广度优先/最佳优先 |
| **最优解** | ✅ 保证 | ✅ 保证 | ⚠️ 需满足贪心选择性质 | ✅ 保证 | ✅ 保证 |
| **时间复杂度** | 通常 O(n log n) | 通常多项式 | 通常低阶多项式 | 通常指数级 | 通常指数级 |
| **空间复杂度** | 递归栈 | DP表 | O(1) 或 O(n) | 递归栈 | 活节点队列 |
| **典型问题** | 归并/快排 | 背包/LCS | MST/Huffman | n皇后/TSP | TSP/装载 |

## 10.2 如何选择算法策略

```
面对一个新问题，按以下流程选择策略：

1. 问题是否有最优子结构？
   ├── 否 → 回溯法 或 分支限界法（穷举+剪枝）
   └── 是 → 继续判断
          │
          ├── 子问题相互独立？
          │   └── 是 → 分治法
          │
          ├── 贪心选择性质成立？
          │   └── 是 → 贪心算法
          │
          └── 子问题重叠？
              └── 是 → 动态规划
```

### 10.2.1 各类问题推荐策略速查

| 问题类型 | 推荐策略 | 时间复杂度 |
|----------|----------|-----------|
| 排序 | 分治法（归并/快排） | O(n log n) |
| 查找 | 分治法（二分查找） | O(log n) |
| 最优化（重叠子问题） | 动态规划 | O(多项式) |
| 最优化（局部最优推全局） | 贪心算法 | O(n log n) |
| 组合搜索（求所有解） | 回溯法 | O(指数) |
| 组合搜索（求最优解） | 分支限界法 | O(指数) |
| 图的最短路径 | 贪心(Dijkstra)/DP(Floyd) | O(n²)/O(n³) |
| 图的最小生成树 | 贪心(Prim/Kruskal) | O(E log V) |
| NP难问题 | 近似/启发式/回溯/分支限界 | 视规模而定 |

## 10.5 算法设计的关键思维方式

### 10.5.1 问题规约思维

将未知问题**转化为**已知问题。例如：
- 二部图匹配 → 最大流
- 最长递增子序列 → LCS（X 排序后与 X 求 LCS）
- 最小顶点覆盖（二部图） → 最大匹配

### 10.5.2 空间换时间

用额外的存储来减少重复计算：
- 动态规划的 DP 表
- 哈希表加速查找
- 记忆化搜索（Memoization）

### 10.5.3 预处理思维

- 排序后处理
- 前缀和、差分数组
- 稀疏表（Sparse Table）处理 RMQ

### 10.5.4 二分答案

当问题满足**单调性**时，可以对答案进行二分：
- "最大值最小化" → 二分答案 + 贪心验证
- "第 k 大/小" → 二分查找

## 10.6 全书核心公式汇总

### Master 定理

```
T(n) = a·T(n/b) + f(n)

情况1: f(n) = O(n^(log_b a - ε))    → T(n) = Θ(n^(log_b a))
情况2: f(n) = Θ(n^(log_b a))        → T(n) = Θ(n^(log_b a) · log n)
情况3: f(n) = Ω(n^(log_b a + ε))    → T(n) = Θ(f(n))
      且 af(n/b) ≤ cf(n), c < 1
```

### 常用递推关系

| 递推式 | 解 | 对应算法 |
|--------|-----|----------|
| T(n) = T(n/2) + O(1) | Θ(log n) | 二分查找 |
| T(n) = 2T(n/2) + O(n) | Θ(n log n) | 归并排序 |
| T(n) = T(n-1) + O(1) | Θ(n) | 线性递归 |
| T(n) = T(n-1) + O(n) | Θ(n²) | 选择排序递归 |
| T(n) = 2T(n-1) + O(1) | Θ(2ⁿ) | 汉诺塔 |
| T(n) = 3T(n/2) + O(n) | Θ(n^(log₂ 3)) | Karatsuba |
| T(n) = 7T(n/2) + O(n²) | Θ(n^(log₂ 7)) | Strassen |

### Stirling 公式

```
n! ≈ √(2πn) · (n/e)ⁿ

log(n!) = Θ(n log n)
```

## 10.7 结语

算法设计与分析是计算机科学的基石。掌握本文涵盖的五大策略（分治、动态规划、贪心、回溯、分支限界）以及图算法、排序查找等核心内容，你已经具备解决绝大多数计算问题的能力。

**记住几个核心原则**：

1. **理解 > 背诵**：理解算法背后的思想，而不仅仅是代码
2. **分析复杂度**：养成分析时间/空间复杂度的习惯
3. **多写多练**：算法能力来自大量练习，没有捷径
4. **从暴力到优化**：先想暴力解法，再逐步优化
5. **举一反三**：一个问题的解法往往能推广到一类问题

> "The best way to learn algorithms is to implement them."
> — 学习算法的最好方式就是亲手实现它们。
