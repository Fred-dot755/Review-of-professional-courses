# 第4章：内存管理

> 存储器是计算机系统的核心资源之一。内存管理的核心任务是为多道程序的运行提供良好的环境，提高内存利用率，并实现逻辑地址到物理地址的映射。

---

## 4.1 存储系统的层次结构

```
                    ┌──────────┐
         更快      │  寄存器    │  ← CPU内部，ns级
         更小      ├──────────┤
         更贵      │   Cache   │  ← SRAM，ns级
         ▲        ├──────────┤
         │        │  主存(RAM) │  ← DRAM，μs级
         │        ├──────────┤
         │        │  磁盘缓存   │
         ▼        ├──────────┤
         更慢      │  磁  盘   │  ← ms级
         更大      ├──────────┤
         更便宜     │  光盘/磁带  │  ← 秒级
                    └──────────┘
```

> **局部性原理**是存储层次结构的基础。

---

## 4.2 程序的装入与链接

### 4.2.1 程序执行的三个步骤

| 步骤 | 功能 | 产物 |
|------|------|------|
| **编译** | 源程序 → 目标模块(机器码) | `.obj` / `.o` |
| **链接** | 多个目标模块 + 库 → 装入模块 | `.exe` |
| **装入** | 装入模块 → 内存，可执行 | 进程 |

### 4.2.2 链接方式

| 方式 | 时机 | 说明 |
|------|------|------|
| **静态链接** | 装入前 | 所有目标模块链接成一个完整文件 |
| **装入时动态链接** | 装入时 | 装入一个模块时发现需其他模块，边装边链接 |
| **运行时动态链接** | 运行时 | 用到时才链接（如 DLL / .so） |

### 4.2.3 装入方式

| 方式 | 说明 | 特点 |
|------|------|------|
| **绝对装入** | 程序中的地址即物理地址 | 单道程序，简单但死板 |
| **可重定位装入** | 装入时将所有地址+偏移量 | 地址变换在装入时一次完成 |
| **动态运行时装入** | 运行时由硬件（MMU）将逻辑地址转物理地址 | 灵活，现代OS使用 |

## 4.3 连续分配存储管理 ⭐

### 4.3.1 单一连续分配

```
┌────────────┐
│   OS区     │
├────────────┤
│   用户区    │  ← 一次只能装一个用户程序
│   (空闲)    │
└────────────┘
```

**特点**：最简单，仅用于单用户单任务OS，内存利用率极低。

### 4.3.2 固定分区分配

> 将内存分为若干大小相等或不等固定的分区。

```
┌────────────┐
│   OS区     │
├────────────┤
│  分区1 4MB  │  ← 作业A(3MB) → 浪费1MB（内部碎片）
├────────────┤
│  分区2 8MB  │  ← 作业B(7MB) → 浪费1MB
├────────────┤
│  分区3 8MB  │  ← 空闲
├────────────┤
│  分区4 16MB │  ← 作业C(16MB)
└────────────┘
```

| 特点 | 说明 |
|------|------|
| **内部碎片** | 分区内未被利用的空间 |
| **外部碎片** | 无法被任何作业使用的空闲分区（太小） |
| **优点** | 简单 |
| **缺点** | 分区大小难以确定，利用率低 |

### 4.3.3 动态分区分配（可变分区）⭐

> 根据进程实际需求，动态分配内存空间。

```
动态分区示例：

初始：|████  OS  ████|_____________ 空闲 ______________|

分配A(20MB)：|████  OS  ████|████  A(20MB)  ████|_______ 空闲 ________|

分配B(50MB)：|██  OS  ██|██  A  ██|██████  B(50MB)  ██████|___ 空闲 ___|

释放A：|██  OS  ██|__ 空闲(20) __|█████  B  █████|___ 空闲 ___|

此时产生外部碎片：20MB的空洞在两个分区之间
```

#### 动态分区分配算法 ⭐

| 算法 | 策略 | 优点 | 缺点 |
|------|------|------|------|
| **首次适应（FF）** | 找到第一个足够大的空闲分区 | 简单、快速 | 低地址端产生小碎片 |
| **循环首次适应（NF）** | 从上一次找到的位置继续搜索 | 空闲分区分布均匀 | 缺乏大空闲分区 |
| **最佳适应（BF）** | 找最接近需求大小的分区 | 保留大分区 | 产生极小的难以利用的碎片 |
| **最坏适应（WF）** | 找最大的空闲分区 | 减少小碎片 | 缺乏大空闲分区 |

```
首次适应（First Fit）图示：

空闲分区链：(10MB) → (4MB) → (20MB) → (18MB) → (7MB)

分配 12MB：
  FF：查找到20MB处，分配后：(10MB)→(4MB)→(8MB)→(18MB)→(7MB)
  BF：查找到12MB最佳匹配→18MB分配后：(10MB)→(4MB)→(20MB)→(6MB)→(7MB)
  WF：查找到20MB最大，分配后：(10MB)→(4MB)→(8MB)→(18MB)→(7MB)
```

### 4.3.4 碎片与紧凑

| 技术 | 说明 |
|------|------|
| **紧凑（Compaction）** | 移动内存中的进程，将小空闲分区合并成大块 |
| **条件** | 必须是可重定位的（动态运行时装入） |
| **代价** | CPU时间消耗大 |

## 4.4 对换（Swapping）

> 将阻塞状态的进程整个换出到外存（对换区），将就绪进程换入内存。

```
内存                          对换区（磁盘）
┌───────────┐   换出    ┌───────────────┐
│  进程 A    │ ───────→ │   A（阻塞态）   │
│  (阻塞)    │          └───────────────┘
└───────────┘
      ↓
┌───────────┐   换入    ┌───────────────┐
│  进程 B    │ ←─────── │   B（就绪态）   │
│  (就绪)    │          └───────────────┘
└───────────┘
```

| 关键点 | 说明 |
|--------|------|
| **对换的时机** | 内存空间紧张时；进程阻塞时 |
| **对换的目标** | 提高内存利用率和CPU利用率 |
| **对换区** | 磁盘上专门用于换入换出的区域，连续分配，速度快 |

## 4.5 基本分页存储管理 ⭐⭐⭐

### 4.5.1 分页的基本概念

```
逻辑地址空间：                    物理内存：
┌─────┐  页号0                  ┌─────────┐
│页面0│                         │  页框0   │
├─────┤  页号1       页表       ├─────────┤
│页面1│             ┌──┬──┐     │  页框1   │
├─────┤             │0 │2 │     ├─────────┤
│页面2│             │1 │4 │     │  页框2   │
├─────┤             │2 │1 │     ├─────────┤
│  ...│             │..│..│     │  页框3   │
└─────┘             └──┴──┘     ├─────────┤
                                │  页框4   │
 页（Page）= 逻辑空间            ├─────────┤
 页框（Frame）= 物理空间          │   ...    │
                                └─────────┘
```

| 概念 | 说明 |
|------|------|
| **页（Page）** | 逻辑地址空间的基本单位 |
| **页框/页帧（Frame）** | 物理内存的基本单位 |
| **页表（Page Table）** | 页号 → 页框号的映射表 |
| **页面大小** | 通常为 4KB（$2^{12}$） |

### 4.5.2 地址变换 ⭐⭐⭐

```
逻辑地址 A 的结构：
┌─────────────────────┬──────────────────────┐
│     页号 P           │      页内偏移 d        │
└─────────────────────┴──────────────────────┘

地址变换过程：
 ① 从逻辑地址A提取页号P和页内偏移d
 ② 查页表，获得P对应的页框号f
 ③ 物理地址 = f × 页面大小 + d
```

```
地址变换示意图：

逻辑地址 → │ 页号P │ 偏移d │
              │
              ▼
    ┌───────────────────┐
    │  页表寄存器(PTBR)   │  ← 页表起始地址
    └────────┬──────────┘
             ▼
    ┌───────────────────┐
    │      页  表        │
    │  ...│  f  │...    │  ← 页号P → 页框号f
    └────────┬──────────┘
             ▼
    物理地址 → │ 页框f │ 偏移d │
```

### 4.5.3 页表项结构

```
页表项：┌──────┬─────┬───┬───┬───────┬───────┐
        │页框号 │ 存在位│读写│脏位│访问位 │保护位  │
        └──────┴─────┴───┴───┴───────┴───────┘

存在位(P)：该页是否在内存中（1=在，0=不在→缺页中断）
读写位(R/W)：该页是否可写
脏位(D)：该页在内存中是否被修改过（1=修改过，换出时需写回）
访问位(A)：该页最近是否被访问过（用于页面置换算法）
```

### 4.5.4 多级页表 ⭐

> 问题：对于64位系统，一级页表过大（如32位地址空间，4KB页面，页表需4MB）。

```
两级页表结构：

逻辑地址：│ 一级页号P1 │ 二级页号P2 │ 页内偏移 d │

    外层页表              内层页表
    ┌──────┐            ┌──────┐
    │ P1→  │ ───────→   │ P2→  │ ──→ 物理页框
    │      │            │      │
    │ P1'→ │ ──→ null   │      │
    └──────┘    (不存在)  └──────┘
```

| 优点 | 说明 |
|------|------|
| **节省内存** | 仅需为实际使用的页建立页表项 |
| **分页存储页表** | 页表本身不需要连续存储 |
| **按需分配** | 内层页表在需要时才创建 |

### 4.5.5 反置页表（Inverted Page Table）

> 传统页表：每个进程一个，按逻辑页号索引。
> 反置页表：整个系统只有一个，按物理页框号索引。

| 对比 | 传统页表 | 反置页表 |
|------|----------|----------|
| **索引方式** | 页号 → 页框号 | 页框号 → (进程ID, 页号) |
| **数量** | 每个进程一个 | 系统只有一个 |
| **大小** | 与进程空间成正比 | 与物理内存大小成正比 |
| **查找方式** | 直接索引 | 需搜索（用哈希表加速） |

## 4.6 基本分段存储管理 ⭐

### 4.6.1 分段的基本概念

> 将用户程序按逻辑意义（代码段、数据段、栈段、堆）划分为若干段，以段为单位进行内存分配。

```
用户程序：
┌────────────────┐
│  代码段（MAIN）  │  段号0，段长+基址
├────────────────┤
│  数据段（DATA）  │  段号1，段长+基址
├────────────────┤
│  栈段（STACK）   │  段号2，段长+基址
├────────────────┤
│  堆段（HEAP）    │  段号3，段长+基址
└────────────────┘
```

### 4.6.2 地址变换

```
逻辑地址：│ 段号 S │ 段内偏移 d │

    ┌──────────────────┐
    │   段表寄存器        │
    │ 段表基址 │ 段表长度  │
    └────────┬──────────┘
             ▼
    ┌──────────────────────────┐
    │        段    表           │
    │ 段号 │ 段长 │ 基址 │ 保护 │
    │  0  │ 1KB  │ 8K  │ r-x │
    │  1  │ 500B │ 12K │ rw- │
    │  2  │ 2KB  │ 20K │ rw- │
    └──────────────────────────┘
             │
             ▼ 基址 + 偏移d = 物理地址
             （若 d < 段长，则合法；否则越界中断）
```

### 4.6.3 分页与分段的对比 ⭐

| 对比项 | 分页 | 分段 |
|--------|------|------|
| **划分依据** | 物理（固定大小，系统决定） | 逻辑（不定长，程序员决定） |
| **目的** | 提高内存利用率 | 满足用户编程需求 |
| **地址维度** | 一维（单一连续地址空间） | 二维（段号+段内偏移） |
| **单位大小** | 固定 | 可变 |
| **共享保护** | 困难（一页可能含多个逻辑单元） | 容易（段是逻辑整体） |
| **碎片** | 内部碎片 | 外部碎片 |

## 4.7 段页式存储管理 ⭐

> 结合分页和分段的优点：先按逻辑分段，每段再按物理分页。

```
逻辑地址结构：│ 段号 S │ 页号 P │ 页内偏移 d │

地址变换流程：
  段号S → 段表[S] → 页表基址
                    ↓
              页号P → 页表[P] → 页框号f
                                ↓
                          物理地址 = f × 页大小 + d
```

```
┌─────────┐     ┌──────────────┐     ┌──────────────┐
│ 段表寄存器 │     │     段表      │     │    页表       │
│ 段表基址  │ ──→ │ 段0→页表0基址  │ ──→ │ 页0→页框5    │
└─────────┘     │ 段1→页表1基址  │     │ 页1→页框8    │
                │ 段2→页表2基址  │     │ 页2→页框3    │
                └──────────────┘     └──────────────┘
```

| 优点 | 说明 |
|------|------|
| **兼具分段的分页的优点** | 逻辑清晰 + 无外部碎片 |
| **易于共享和保护** | 以段为单位 |
| **缺点** | 每次访问需三次访存（段表→页表→数据），开销大 |

## 4.8 装入程序（Loader）相关补充

### 4.8.1 装入时重定位 vs 运行时重定位

| 重定位类型 | 时机 | 特点 |
|-----------|------|------|
| **静态重定位（装入时）** | 程序装入内存时一次性完成 | 装入后地址不变，不可移动 |
| **动态重定位（运行时）** | 每次访存时由MMU完成 | 支持紧凑、虚拟存储、分页分段 |

### 4.8.2 MMU（内存管理单元）

> MMU是CPU中负责地址转换的硬件部件，将虚拟地址转换为物理地址。

```
CPU发出逻辑地址 → MMU → 物理地址 → 内存

若缺页：MMU触发缺页中断 → OS处理
```

## ⭐ 本章小结

| 知识点 | 核心内容 |
|--------|----------|
| **存储层次** | 寄存器 → Cache → 内存 → 磁盘 → 磁带 |
| **连续分配** | 单一分区、固定分区、动态分区（FF/BF/WF/NF） |
| **离散分配** | 分页、分段、段页式 |

| **紧凑** | 移动进程合并外部碎片，需动态重定位 |
| **对换** | 进程整体换入换出，平衡内存负载 |
