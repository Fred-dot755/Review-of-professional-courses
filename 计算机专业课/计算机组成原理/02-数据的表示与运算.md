# 第2章 数据的表示与运算

> 计算机内部一切信息都以二进制形式表示。本章系统讲解数值数据和非数值数据在计算机中的表示方法，以及基本运算的实现原理。这是理解计算机底层运作机制的基石。

---

## 2.1 进位计数制及其转换

### 2.1.1 常用进位计数制

| 进制 | 基数 | 数码 | 后缀/前缀 |
|------|------|------|-----------|
| 二进制（Binary） | 2 | 0, 1 | `B` 或 `0b` |
| 八进制（Octal） | 8 | 0~7 | `O` 或 `0` |
| 十进制（Decimal） | 10 | 0~9 | `D`（可省略） |
| 十六进制（Hexadecimal） | 16 | 0~9, A~F | `H` 或 `0x` |

### 2.1.2 任意进制转十进制 —— 按权展开法

对于一个 R 进制数，每位上的数码 K 乘以该位的权值（即 R 的幂次），然后全部加起来，就得到十进制值。

```
十进制值 = (整数部分各位 × R的正幂次) + (小数部分各位 × R的负幂次)

具体来说，对于 R 进制数：K_n K_{n-1} ... K_1 K_0 . K_{-1} K_{-2} ... K_{-m}

  = K_n×R^n + K_{n-1}×R^{n-1} + ... + K_0×R^0 + K_{-1}×R^{-1} + ... + K_{-m}×R^{-m}

其中 K_i 是第 i 位上的数码（0 ~ R-1），i 越大权重越大。
```

> **示例**：(1011.01)₂ 转十进制：
>
>   = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ + 0×2⁻¹ + 1×2⁻²
>   = 1×8 + 0×4 + 1×2 + 1×1 + 0×0.5 + 1×0.25
>   = 8 + 0 + 2 + 1 + 0 + 0.25
>   = (11.25)₁₀

### 2.1.3 十进制转任意进制

将一个十进制数转换为 R 进制数时，需要**整数部分**和**小数部分**分开处理，最后再拼接起来。

#### 整数部分 —— 除 R 取余法（倒序排列）

**口诀**：反复除以 R，记下每次的**余数**，直到商为 0，然后把余数从下往上（从后往前）倒着读。

**步骤**：

1. 用十进制整数除以基数 R，得到**商**和**余数**。
2. 记录余数（这就是 R 进制数的一位）。
3. 用上一步的商继续除以 R，重复直到**商为 0**。
4. 将所有余数**从后往前（倒序）排列**，即为结果的整数部分。

> **示例**：将 (53)₁₀ 转换为二进制（R=2）
>
> | 步骤 | 运算 | 商 | 余数 | 说明 |
> |------|------|----|------|------|
> | ① | 53 ÷ 2 | 26 | **1** | ← 最低位（最右边）|
> | ② | 26 ÷ 2 | 13 | **0** | |
> | ③ | 13 ÷ 2 | 6  | **1** | |
> | ④ | 6 ÷ 2  | 3  | **0** | |
> | ⑤ | 3 ÷ 2  | 1  | **1** | |
> | ⑥ | 1 ÷ 2  | 0  | **1** | ← 最高位（最左边）|
>
> **倒序排列余数**（从下往上读）：`110101`
>
> 结果：(53)₁₀ = (110101)₂

#### 小数部分 —— 乘 R 取整法（顺序排列）

**口诀**：反复乘以 R，记下每次的**整数部分**，把整数部分从上往下（正序）读。

**步骤**：

1. 用十进制小数乘以基数 R，得到**积**。
2. 取出积的**整数部分**（这就是 R 进制的一位小数）。
3. 用积的**小数部分**继续乘以 R，重复。
4. 将每次取出的整数部分**按顺序排列**（从上往下），即为结果的小数部分。

**终止条件**：小数部分为 0（精确转换）或达到所需精度位数。

> **示例**：将 (0.6875)₁₀ 转换为二进制（R=2）
>
> | 步骤 | 运算 | 积 | 取整 | 说明 |
> |------|------|----|------|------|
> | ① | 0.6875 × 2 | 1.375 | **1** | ← 最高小数位（紧挨小数点）|
> | ② | 0.375 × 2  | 0.75  | **0** | |
> | ③ | 0.75 × 2   | 1.5   | **1** | |
> | ④ | 0.5 × 2    | 1.0   | **1** | ← 最低小数位（最右边）|
>
> ④ 之后小数部分为 0.0，转换结束。
>
> **顺序排列整数部分**（从上往下读）：`1011`
>
> 结果：(0.6875)₁₀ = (0.1011)₂

#### 完整转换：整数 + 小数一起转

将整数部分和小数部分分别转换后，用小数点拼接即可。

> **综合示例**：将 (53.6875)₁₀ 转换为二进制
>
> 整数部分：(53)₁₀ = (110101)₂
> 小数部分：(0.6875)₁₀ = (0.1011)₂
>
> **拼接结果**：(53.6875)₁₀ = (110101.1011)₂

> **⚠️ 重要提示**：并非所有十进制小数都能用二进制**精确**表示。例如 (0.3)₁₀ 转换为二进制是无限循环小数 0.01001001001...（循环节 01001），这在编程中会造成精度丢失（如 `0.1 + 0.2 != 0.3` 的现象）。

### 2.1.4 二、八、十六进制间的快速转换

| 二进制 ↔ 八进制 | 3位一组 |
| 二进制 ↔ 十六进制 | 4位一组 |

```
二进制 1011 0110 . 1101 0100
         ↓            ↓
十六进制  B    6   .   D    4  = (B6.D4)₁₆
```

## 2.2 BCD码与字符编码

### 2.2.1 BCD码（Binary-Coded Decimal）

用4位二进制数表示1位十进制数的编码方式。

| BCD码类型 | 特点 |
|-----------|------|
| **8421码**（有权码） | 最常用，四位权值分别为 8、4、2、1 |
| **余3码**（无权码） | 8421码 + 0011，是一种自补码 |
| **2421码**（有权码） | 权值为 2、4、2、1，也是一种自补码 |

### 2.2.2 ASCII码

**ASCII（American Standard Code for Information Interchange）** 是国际上最通用的字符编码系统，共128个字符。

| 范围 | 字符类型 | 记忆要点 |
|------|----------|----------|
| 0~31 | 控制字符 | 不可打印 |
| 32 | 空格（SPACE） | — |
| 48~57 | 数字 `0~9` | `'0'` = 48 |
| 65~90 | 大写字母 `A~Z` | `'A'` = 65 |
| 97~122 | 小写字母 `a~z` | `'a'` = 97 |

> **关键规律**：大写字母和小写字母相差32（即 `'a' - 'A' = 32`），ASCII码中大小写字母仅第5位（从0计数）不同。

### 2.2.3 Unicode 与 UTF-8

- **Unicode**：通用字符集，为世界上所有字符分配唯一码点。
- **UTF-8**：Unicode的变长实现方式，1~4字节编码，向后兼容ASCII。

## 2.3 机器数的定点表示

### 2.3.1 机器数与真值

| 概念 | 说明 |
|------|------|
| **真值** | 日常生活中正负号加绝对值的写法，如 `+1011`、`-1101` |
| **机器数** | 将符号数字化（0表示正，1表示负）后存储在计算机中的数 |

### 2.3.2 原码（Sign-Magnitude）

- **定义**：最高位为符号位（0正1负），其余位为数值的绝对值。
- n 位原码的范围：-(2^(n-1)-1) ～ +(2^(n-1)-1)

> **示例（8位）**：
> [+13]原 = 00001101
> [-13]原 = 10001101
> [+0]原 = 00000000，[-0]原 = 10000000（存在 ±0）

### 2.3.3 反码（Ones' Complement）

- **定义**：正数的反码与原码相同；负数的反码是符号位不变，其余各位按位取反。
- n 位反码的范围：-(2^(n-1)-1) ～ +(2^(n-1)-1)

### 2.3.4 补码（Two's Complement）⭐

> 补码是计算机中最常用的数值表示方法，因为它可以将减法统一为加法运算。

- **定义**：正数的补码与原码相同；负数的补码是反码末位+1（符号位不变）。
- **等价定义**：[X]补 = 2^n + X （模 2^n）
- n 位补码的范围：-2^(n-1) ～ +(2^(n-1)-1)

```
快速求补码口诀（负数）：
从右往左找到第一个1，该位左侧各位取反，该位及右侧保持不变。
```

> **补码的关键优势**：
> - 零的表示唯一：[+0]补 = [-0]补 = 00000000
> - 比原码/反码多表示一个负数：[-128]补 = 10000000（8位）
> - 减法用加法实现：[A-B]补 = [A]补 + [-B]补

### 2.3.5 移码（Biased Representation）

- **定义**：[X]移 = 2^(n-1) + X（在真值上加一个偏移量）
- 常用于浮点数的阶码表示
- 移码和对应补码的符号位刚好相反

## 2.4 定点数的运算

### 2.4.1 定点加法与减法

计算机中使用补码进行加减运算的核心公式：

```
[A+B]补 = [A]补 + [B]补   (模 2^n)
[A-B]补 = [A]补 + [-B]补  (模 2^n)
```

#### 溢出判断的三种方法

| 方法 | 判断条件 | 原理 |
|------|----------|------|
| **单符号位法** | 两个操作数符号相同但与结果符号不同 | 逻辑判断 |
| **双符号位（变形补码）法** | 双符号位分别为 S_s1 和 S_s2，异号时溢出 | S_s1 ⊕ S_s2 = 1 |
| **进位判断法** | 符号位的进位 C_n 与最高数值位的进位 C_(n-1) 不同 | C_n ⊕ C_(n-1) = 1 |

### 2.4.2 定点乘法

#### 原码一位乘法

1. 符号位单独处理：P_s = X_s ⊕ Y_s（异或）
2. 数值部分为绝对值相乘
3. 通过移位和加法迭代完成

#### 补码一位乘法（Booth算法）⭐

Booth乘法的核心规则：

| Y_i | Y_(i+1) | 操作 |
|-----|---------|------|
| 0   | 0       | 部分积右移1位 |
| 0   | 1       | 部分积 + [X]补，右移1位 |
| 1   | 0       | 部分积 + [-X]补，右移1位 |
| 1   | 1       | 部分积右移1位 |

> 其中 Y_i 和 Y_(i+1) 代表乘数中相邻两位（Y_n = 0 作为初始附加位）。Booth算法通过一次处理两位，减少了部分积的累加次数。

### 2.4.3 定点除法

- **原码恢复余数除法**：通过反复减除数判断是否够减，不够减时需恢复余数。
- **原码加减交替除法（不恢复余数法）**：效率更高的除法算法。够减则商1并左移除数；不够减则商0并加上除数再左移。

## 2.5 浮点数的表示与运算

### 2.5.1 浮点数的表示格式

一个浮点数 N 的一般形式为：

```
N = M × R^E
```

| 符号 | 名称 | 含义 |
|------|------|------|
| M | 尾数（Mantissa） | 定点小数，用原码或补码表示 |
| E | 阶码（Exponent） | 定点整数，用移码表示 |
| R | 基数（Radix） | 通常为2（隐含） |

```
浮点数的机器格式：

┌──────┬──────────┬─────────────────────┐
│  S   │     E    │          M          │
│ 数符  │   阶码    │     尾数（绝对值）    │
│ 1位  │   k位     │        n位          │
└──────┴──────────┴─────────────────────┘
```

### 2.5.2 IEEE 754 标准 ⭐

这是现代计算机中最重要的浮点数标准，几乎所有CPU和编程语言都遵循它。

#### 基本格式

| 类型 | 总位数 | 符号位 | 阶码位 | 尾数位 | 偏置值 |
|------|--------|--------|--------|--------|--------|
| **单精度（float）** | 32 | 1 | 8 | 23 | 127 |
| **双精度（double）** | 64 | 1 | 11 | 52 | 1023 |

#### 关键规定

1. **尾数用原码**表示，采用隐藏位技术（规格化数的尾数整数部分隐含为1，不存储）。
2. **阶码用移码**表示，偏移量（bias）为 2^(k-1) - 1。

#### IEEE 754 特殊值

以单精度为例：

| 阶码E | 尾数M（小数部分） | 表示的值 |
|-------|------------------|----------|
| E=0, M=0 | 全0 | ±0 |
| E=0, M≠0 | 非0 | 非规格化数 |
| 1 ≤ E ≤ 254 | 任意 | 规格化数: (-1)^S × 1.M × 2^(E-127) |
| E=255, M=0 | 全0 | ±∞ |
| E=255, M≠0 | 非0 | NaN (Not a Number) |

> **实用换算示例**：将十进制数 -12.625 转换为单精度浮点数
>
> - 12.625 = 1100.101₂ = 1.100101₂ × 2³
> - 数符 S=1（负数）
> - 阶码 E = 3 + 127 = 130 = 10000010₂
> - 尾数 M = 10010100000000000000000（隐藏首位1）
> - 最终：`1 10000010 10010100000000000000000`

### 2.5.3 浮点数的规格化

**规格化**的目的是使尾数的最高位为1，保证最多的有效位用于表示精度。

| 类型 | 规格化形式 |
|------|-----------|
| 原码规格化 | 尾数最高数值位为1（即 1/2 ≤ |M| < 1） |
| 补码规格化 | 正数时最高位为1；负数时最高位为0（即 -1/2 ≤ M补 < -1 或 1/2 ≤ M补 < 1） |

> **IEEE 754的隐藏位**：规格化尾数的最高位恒为1，因此可以不存储，多出1位精度。

### 2.5.4 浮点数的四则运算

#### 浮点加减法步骤

```
1. 对阶（小阶向大阶看齐）→ 阶码统一
2. 尾数相加/减
3. 结果规格化（可能需要右规或左规）
4. 舍入处理
5. 溢出判断
```

#### 舍入方法

| 方法 | 说明 |
|------|------|
| **截断法** | 直接丢弃多余位（最粗糙） |
| **恒置1法** | 无论丢弃部分为何，保留部分末位置1 |
| **0舍1入法** | 类似十进制四舍五入 |
| **IEEE 754默认：就近舍入（Round to Nearest, Ties to Even）** | 最精确的方式，向最接近的可表示值舍入；当处于正中时选偶数 |

## 2.6 数据校验码

### 2.6.1 奇偶校验码

| 类型 | 定义 |
|------|------|
| **奇校验** | 使整个码字中1的个数为奇数 |
| **偶校验** | 使整个码字中1的个数为偶数 |

> 奇偶校验只能检测奇数个错误，**无法纠错**，也无法检测偶数个错误。常用于对成本敏感的存储和传输场景。

### 2.6.2 海明码（Hamming Code）

海明码是一种**能纠错**的编码方案。通过设置多个校验位，不仅能检测错误，还能定位错误位。

- 对于 k 个数据位，需要 r 个校验位满足：2^r ≥ k + r + 1
- 海明码的码距（最小不同位数）为3，可纠正1位错、检测2位错。

### 2.6.3 循环冗余校验码（CRC）

广泛应用于网络通信和存储设备。基于模2除法实现，发送方在数据末尾附加校验码，接收方通过除法运算验证。

## 本章小结

本章涵盖了计算机中数据表示和运算的核心内容，重点总结：

| 知识点 | 核心要点 |
|--------|----------|
| **补码** | 计算机中最核心的数值表示，实现加减法统一，0的表示唯一 |
| **IEEE 754** | 现代浮点数的通用标准，掌握单精度和双精度的格式与转换 |
| **溢出判断** | 双符号位法、进位判别法是考试和实践中常用的方法 |
| **Booth乘法** | 补码一位乘法的经典算法 |
| **海明校验** | 能纠错1位的编码方案，理解其分组校验原理 |
| **浮点加减法** | 对阶 → 尾数运算 → 规格化 → 舍入 → 判溢出 |

理解这些底层表示和运算原理，有助于写出更高效、更可靠的程序，也能帮助理解编程语言中各种数值类型的行为（如整数溢出、浮点数精度问题等）。
