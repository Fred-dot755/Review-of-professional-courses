# 第7章：磁盘存储器与I/O系统

> I/O系统是操作系统中最繁杂的部分，管理着各种速度迥异、特性不同的I/O设备。本章系统介绍I/O系统层次结构、磁盘调度算法、I/O控制方式及设备管理核心技术。

---

## 7.1 I/O系统概述

### 7.1.1 I/O系统层次结构

```
        用户层I/O软件（库函数）
              │
              ▼
        设备独立性软件（文件系统层）
              │
              ▼
        设备驱动程序（驱动层）
              │
              ▼
        中断处理程序
              │
              ▼
        硬件设备
```

| 层次 | 功能 | 示例 |
|------|------|------|
| **用户层I/O** | 格式化I/O，提供API | `printf()`, `scanf()` |
| **设备独立性软件** | 统一命名、保护、缓冲、分配 | 文件系统逻辑层 |
| **设备驱动程序** | 将通用命令转为设备特定指令 | 磁盘驱动、显卡驱动 |
| **中断处理程序** | 处理I/O完成中断 | 中断服务例程 |

### 7.1.2 I/O设备分类

```
I/O设备分类：
    │
    ├── 按传输速率
    │    ├── 低速设备（键盘/鼠标：B/s）
    │    ├── 中速设备（打印机：KB/s）
    │    └── 高速设备（磁盘：MB/s~GB/s）
    │
    ├── 按信息交换单位
    │    ├── 块设备（磁盘：以块为单位，可寻址）
    │    └── 字符设备（键盘/打印机：以字符流为单位）
    │
    └── 按共享属性
         ├── 独占设备（打印机：一次一个进程使用）
         ├── 共享设备（磁盘：多进程可同时使用）
         └── 虚拟设备（SPOOLing）
```

## 7.2 I/O控制方式 ⭐⭐⭐

> 四种I/O控制方式分别在不同的发展阶段出现，核心演变围绕**减少CPU对I/O的干预**。

### 7.2.1 四种方式总览

```
I/O控制方式演进（CPU干预越来越少，I/O效率越来越高）：

程序直接控制 ──→ 中断驱动 ──→ DMA ──→ I/O通道
  (忙等待)      (CPU响应中断)  (块传输)  (通道程序)
```

| 方式 | 数据传送单位 | CPU干预程度 | 适用场景 |
|------|-------------|-------------|----------|
| **程序直接控制** | 字（字节） | 全程参与（忙等待） | 极简单设备、早期系统 |
| **中断驱动** | 字（字节） | 每次数据传输中断 | 低速字符设备 |
| **DMA** | 块 | 仅开始和结束 | 磁盘等块设备 |
| **I/O通道** | 多块/多设备 | 最少（通道程序执行） | 大型机 |

### 7.2.2 程序直接控制方式

> CPU反复查询设备状态寄存器，"忙等待"直到设备就绪。

```
程序查询流程：

① CPU向控制器发I/O命令
      │
      ▼
② 反复读状态寄存器
      │
   ┌──┴── busy?
   │
   │ 是 → 继续查询（忙等）
   │
   └── 否 → 就绪
         │
         ▼
③ CPU读/写数据寄存器
      │
      ▼
④ 传输完成，继续下一字节

问题：CPU利用率极低，CPU每次传输都要等待设备
```

### 7.2.3 中断驱动方式

> CPU发I/O命令后继续执行其他任务，设备就绪时发中断通知CPU。

```
中断驱动流程：

① CPU → 发I/O命令 → 设备控制器
② CPU继续执行其他进程（不等待！）
③ 设备准备数据...
④ 设备就绪 → 发中断信号
⑤ CPU响应中断，转入中断服务程序
⑥ 中断服务程序：读数据寄存器，唤醒等待进程
⑦ 返回被中断的进程
```

| 优点 | 缺点 |
|------|------|
| CPU无需忙等待 | 每次传输一个字/字节，中断次数多 |
| 支持多设备并发 | 中断服务开销大（保存/恢复现场） |

### 7.2.4 DMA（直接存储器访问）⭐⭐⭐

> **DMA控制器**取代CPU负责内存与外设之间的**整块数据**传输。

```
DMA控制方式：

        ┌─────────┐
        │   CPU   │
        └────┬────┘
             │ ① 发DMA命令（内存地址+字节数+方向）
             ▼
        ┌─────────┐     ② 数据传输（总线控制权交给DMA）
        │   DMA   │ ←──────────→  设备控制器 ←→ I/O设备
        │ 控制器   │       (窃取总线周期)
        └────┬────┘
             │ ③ 传输完成 → 发中断通知CPU
             ▼
        ┌─────────┐
        │   CPU   │  ← ④ CPU处理完成中断（后处理）
        └─────────┘
```

| 特性 | 说明 |
|------|------|
| **传送单位** | 数据块（不是字/字节） |
| **数据方向** | 内存 ↔ I/O设备（不经CPU） |
| **控制权** | DMA控制器获取总线控制权（周期挪用） |
| **中断** | 每块完成发一次中断（而非每字） |
| **物理地址** | DMA通常使用物理地址（非虚拟地址） |

#### DMA与中断驱动对比

| 对比项 | 中断驱动 | DMA |
|--------|----------|-----|
| **数据单位** | 字/字节 | 块 |
| **中断频率** | 每字一次 | 每块一次 |
| **CPU参与** | 每字传输CPU需中断处理 | 仅开始和结束 |
| **适用** | 低速字符设备 | 高速块设备 |

### 7.2.5 I/O通道

> **通道**是有自己的指令和程序的**专门I/O处理器**，可执行**通道程序**来控制多台设备、多块数据的传输。

| 类型 | 说明 |
|------|------|
| **字节多路通道** | 分时为多个低速设备服务 |
| **数组选择通道** | 独占通道为高速设备传输成组数据 |
| **数组多路通道** | 结合上述两者优点 |

## 7.3 缓冲技术 ⭐

### 7.3.1 引入缓冲的目的

| 目的 | 说明 |
|------|------|
| **缓和CPU与I/O速度不匹配** | CPU快，I/O慢，缓冲吸收速度差 |
| **减少对CPU的中断频率** | 一次缓冲满再发中断 |
| **提高CPU与I/O的并行性** | CPU处理缓冲区数据时，I/O可继续准备下一批 |
| **解决数据粒度不匹配** | 生产者和消费者的数据单位不一致 |

### 7.3.2 缓冲类型

```
缓冲演进：

单缓冲 ──→ 双缓冲 ──→ 循环缓冲 ──→ 缓冲池
(简单)     (乒乓)     (环形)     (通用)
```

#### (1) 单缓冲

```
进程 ──→ [缓冲区] ──→ I/O设备

时间序列：设备填充缓冲 → 进程取走数据 → 设备继续填充...
缺点：设备和进程交替使用缓冲区，并行度低
```

#### (2) 双缓冲（乒乓缓冲）

```
进程 ──→ [缓冲A] ──→ I/O设备
  └──→ [缓冲B] ───↗

并行：设备填充缓冲A时，进程可处理缓冲B的数据
```

#### (3) 循环缓冲

```
多缓冲环：
    [buf1] → [buf2] → [buf3] → [buf4]
       ↑                          │
       └──────────────────────────┘

设备写指针：指向正在填充的缓冲
进程读指针：指向正在处理的缓冲
```

#### (4) 缓冲池 ⭐

```
缓冲池：
┌────────────────────────────────────┐
│  空闲队列：[buf_a]→[buf_b]→[buf_c] │
├────────────────────────────────────┤
│  输入队列：[buf_d]→[buf_e]         │  ← 装满输入数据的缓冲
├────────────────────────────────────┤
│  输出队列：[buf_f]→[buf_g]         │  ← 装满待输出数据的缓冲
└────────────────────────────────────┘

操作：
  - 收容输入：空闲缓冲 → 装满数据 → 进入输入队列
  - 提取输入：输入队列 → 提取数据 → 进入空闲队列
  - 收容输出：空闲缓冲 → 装满输出数据 → 进入输出队列
  - 提取输出：输出队列 → 写回设备 → 进入空闲队列
```

## 7.4 设备分配与SPOOLing ⭐⭐

### 7.4.1 设备分配

```
设备分配涉及的数据结构：

SDT（系统设备表）
  ↓
DCT（设备控制表）  ← 每个设备一个
  ↓
COCT（控制器控制表） ← 每个控制器一个
  ↓
CHCT（通道控制表）  ← 每个通道一个
```

### 7.4.2 设备分配策略

| 策略 | 说明 | 适用 |
|------|------|------|
| **先来先服务** | 按请求顺序分配 | 通用 |
| **优先级高者优先** | 优先级高的进程先获得设备 | 系统设备 |

### 7.4.3 SPOOLing技术 ⭐⭐

> **SPOOLing**（Simultaneous Peripheral Operations On-Line，假脱机技术）将独占设备改造为虚拟共享设备。

```
SPOOLing系统组成：

┌─────────────────────────────────────────┐
│                主机                      │
│  ┌──────────────────────────────────┐   │
│  │          SPOOLing系统             │   │
│  │                                  │   │
│  │   输入进程       输出进程          │   │
│  │    ↓               ↓              │   │
│  │ [输入井(磁盘)]  [输出井(磁盘)]     │   │
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
         │                │
         ▼                ▼
    输入设备              输出设备
    (多台虚拟)            (多台虚拟)
```

| 组成部分 | 功能 |
|----------|------|
| **输入井/输出井** | 磁盘上的模拟输入/输出区 |
| **输入进程** | 将数据从输入设备预取到输入井 |
| **输出进程** | 将输出井中的数据写到输出设备 |
| **井管理程序** | 管理输入输出井 |

> SPOOLing核心思想：用**磁盘**模拟独占设备，使每个用户感觉独占设备。

```
打印机SPOOLing（经典例子）：

进程A → [输出井(A的文件)] ──┐
进程B → [输出井(B的文件)] ──┤ → 输出进程 → 打印机
进程C → [输出井(C的文件)] ──┘   (按队列顺序打印)
```

## 7.5 磁盘存储器管理 ⭐⭐⭐

### 7.5.1 磁盘结构与参数

```
磁盘结构：
        盘面（两面均可读写）
       ┌──────────┐
       │ 磁道(track)│  ← 盘面上同心圆环
       │ 扇区(sector)│ ← 磁道上的弧段（最小读写单位，通常512B）
       └──────────┘
       
       柱面(cylinder)：多盘面同一位置磁道集合
```

```
磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间

T_access = T_seek + T_rotation + T_transfer

寻道时间 T_seek：磁头移动到目标柱面的时间（最慢，ms级）
旋转延迟 T_rotation：目标扇区转到磁头下方的时间（平均半圈）
传输时间 T_transfer：数据在磁盘和内存间的传输时间
```

| 参数 | 典型值 |
|------|--------|
| **转速** | 5400/7200/10000/15000 RPM |
| **平均寻道时间** | 4-10ms |
| **平均旋转延迟** | 7200RPM ≈ 4.16ms（半圈） |
| **扇区大小** | 512B（传统）/ 4KB（高级格式化） |

### 7.5.2 磁盘调度算法 ⭐⭐⭐

> 目标：减少寻道时间（访问时间中最慢的部分）

#### (1) 先来先服务（FCFS）

```
请求队列：98, 183, 37, 122, 14, 124, 65, 67
起始位置：53

FCFS：
53→98→183→37→122→14→124→65→67
总寻道距离：45+85+146+85+108+110+59+2 = 640 柱面
```

| 优点 | 缺点 |
|------|------|
| 公平、简单 | 寻道距离长，性能差 |

#### (2) 最短寻道时间优先（SSTF）⭐

> 选择离当前磁头位置最近的请求。

```
起始：53
SSTF选择顺序：53→65→67→37→14→98→122→124→183
总寻道：12+2+30+23+84+24+2+59 = 236 柱面（远好于640）
```

| 优点 | 缺点 |
|------|------|
| 寻道距离短 | 可能产生**饥饿**（远处请求无限等待） |

#### (3) 扫描算法（SCAN / 电梯算法）⭐

> 磁头单向移动到底，沿途服务，反向再继续。

```
起始：53，方向→（增大方向）

SCAN：53→65→67→98→122→124→183→(反向)→37→14
总寻道：(65-53)+(67-65)+...+(183-122)+(183-37)+(37-14) = 236 柱面
```

| 优点 | 缺点 |
|------|------|
| 无饥饿 | 两端请求等待时间不对称 |

#### (4) 循环扫描算法（C-SCAN）

> 磁头单向移动到底后，**直接跳回起点**（不服务），再开始移动服务。

```
起始：53，方向→

C-SCAN：53→65→67→98→122→124→183→(跳回0,不服务)→14→37
总寻道：(65-53)+...+(183-124)+(183-0)+(37-14)+(37-0)...
```

| 优点 | 缺点 |
|------|------|
| 等待时间更均匀 | 寻道距离可能增加 |

#### (5) LOOK 与 C-LOOK ⭐

> LOOK = SCAN的优化：不需要走到尽头，到最远端即可回头。
> C-LOOK = C-SCAN的优化：到最远端后跳回起点。

```
C-LOOK：53→65→67→98→122→124→183→(跳回14)→37→53方向
        到183不再往前，直接跳回14开始
```

### 7.5.3 磁盘调度算法对比

| 算法 | 寻道性能 | 公平性 | 饥饿 | 实现 |
|------|----------|--------|------|------|
| **FCFS** | 差 | ★★★ | 无 | 最简单 |
| **SSTF** | 好 | ★ | 有 | 中等 |
| **SCAN** | 较好 | ★★ | 无 | 中等 |
| **C-SCAN** | 较好 | ★★★ | 无 | 中等 |
| **C-LOOK** | 好 | ★★★ | 无 | 中等 |

## 7.6 磁盘可靠性技术

### 7.6.1 坏块处理

| 方法 | 说明 |
|------|------|
| **硬件处理** | 磁盘控制器维护坏块列表，自动重映射到备用块 |
| **软件处理** | OS维护坏块文件，避免使用 |

### 7.6.2 RAID技术 ⭐

> **RAID**（Redundant Array of Independent Disks，独立磁盘冗余阵列）：多块磁盘组合提高性能和/或可靠性。

| RAID级别 | 名称 | 技术 | 最少盘数 | 容错 | 空间利用率 |
|----------|------|------|----------|------|-----------|
| **RAID 0** | 条带化 | 数据分条到多块盘 | 2 | 无 | 100% |
| **RAID 1** | 镜像 | 完全复制（镜像） | 2 | 1块故障 | 50% |
| **RAID 4** | 独立校验盘 | 块级条带+专用奇偶校验盘 | 3 | 1块故障 | (N-1)/N |
| **RAID 5** | 分布式校验 | 块级条带+分布式奇偶校验 | 3 | 1块故障 | (N-1)/N |
| **RAID 6** | 双校验 | 两个独立校验方案 | 4 | 2块故障 | (N-2)/N |
| **RAID 10** | 镜像+条带 | 先镜像再条带（RAID 1+0） | 4 | 每组可坏1块 | 50% |

```
RAID 0（条带化）：          RAID 1（镜像）：
Disk0   Disk1              Disk0   Disk1
┌────┐  ┌────┐             ┌────┐  ┌────┐
│ A0 │  │ A1 │             │ A  │  │ A  │  ← 完全复制
│ B0 │  │ B1 │             │ B  │  │ B  │
│ C0 │  │ C1 │             │ C  │  │ C  │
└────┘  └────┘             └────┘  └────┘

RAID 5（分布式校验）：
Disk0   Disk1   Disk2
┌────┐  ┌────┐  ┌────┐
│ A0 │  │ A1 │  │ P_A│  ← 校验分布在所有盘
│ B0 │  │ P_B│  │ B1 │
│ P_C│  │ C0 │  │ C1 │
└────┘  └────┘  └────┘
```

## 7.7 设备独立性

### 7.7.1 逻辑设备与物理设备

> **设备独立性**：应用程序独立于具体物理设备，通过逻辑设备名访问。

```
好处：
  - 增加了设备分配的灵活性
  - 提高了系统的可适应性（设备变化不影响程序）
  - 易于实现I/O重定向
```

### 7.7.2 逻辑设备表（LUT）

| 字段 | 说明 |
|------|------|
| **逻辑设备名** | 程序使用的设备标识 |
| **物理设备名** | 实际设备的标识 |
| **驱动程序入口** | 对应驱动程序的入口地址 |

## ⭐ 本章小结

| 知识点 | 核心内容 |
|--------|----------|
| **I/O控制方式** | 程序直接控制→中断驱动→DMA→I/O通道（CPU干预递减） |
| **DMA** | 块传输，仅开始和结束需CPU干预，周期挪用 |
| **缓冲技术** | 单缓冲→双缓冲→循环缓冲→缓冲池 |

| **磁盘调度算法** | FCFS、SSTF（可能饥饿）、SCAN（电梯）、C-SCAN、C-LOOK |
| **RAID** | RAID 0(条带)、RAID 1(镜像)、RAID 5(分布式校验)、RAID 6(双校验) |
| **设备独立性** | 逻辑设备名→物理设备，LUT映射 |
