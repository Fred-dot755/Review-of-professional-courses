# 第5章 中央处理器（CPU）

> 中央处理器（CPU, Central Processing Unit）是计算机的核心部件，负责指令的执行和数据的处理。本章深入讲解CPU的内部结构、指令执行流程、数据通路设计以及控制器的两种实现方式。CPU是计算机组成原理中最"硬核"的一章。

---

## 5.1 CPU的基本结构

### 5.1.1 CPU的组成

CPU由**运算器**和**控制器**两大部分组成：

```
                          CPU
        ┌──────────────────┴──────────────────┐
        │                                     │
   ┌────┴─────┐                         ┌─────┴─────┐
   │  运算器   │                         │  控制器    │
   │  (ALU)   │                         │  (CU)     │
   └────┬─────┘                         └─────┬─────┘
        │                                     │
   ┌────┴────────────┐              ┌─────────┴──────────┐
   │ ALU（算逻单元）  │              │ PC（程序计数器）    │
   │ ACC（累加器）    │              │ IR（指令寄存器）    │
   │ MQ（乘商寄存器）  │              │ ID（指令译码器）    │
   │ DR（数据寄存器）  │              │ 时序产生器          │
   │ PSW（状态字）     │              │ 微操作信号发生器     │
   └─────────────────┘              └────────────────────┘
```

### 5.1.2 CPU中的主要寄存器

| 寄存器 | 英文名称 | 功能 |
|--------|----------|------|
| **PC** | Program Counter | 存放当前要执行指令的地址，具有"+1"计数功能 |
| **IR** | Instruction Register | 存放当前正在执行的指令 |
| **ID** | Instruction Decoder | 对IR中的操作码进行译码 |
| **MAR** | Memory Address Register | 存放访问主存的地址 |
| **MDR** | Memory Data Register | 存放与主存交换的数据 |
| **ACC** | Accumulator | 累加器，存放ALU的一个操作数或结果 |
| **MQ** | Multiplier-Quotient Register | 乘商寄存器，乘除法专用 |
| **PSW** | Program Status Word | 程序状态字，存放条件码和状态标志 |

### 5.1.3 用户可见 vs 用户不可见寄存器

| 类型 | 包含 | 说明 |
|------|------|------|
| **用户可见寄存器** | 通用寄存器、数据寄存器、地址寄存器 | 程序员可通过指令直接访问 |
| **用户不可见寄存器** | MAR、MDR、IR、ID | 对程序员透明，由CPU内部使用 |

## 5.2 指令周期（Instruction Cycle）⭐

### 5.2.1 指令周期的基本概念

CPU执行一条指令的全过程称为**指令周期**。一个完整的指令周期通常包含：

```
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│ 取指周期 │───▶│ 译码周期 │───▶│ 执行周期 │───▶│ 中断周期 │
│ (Fetch) │    │(Decode)│    │(Execute)│    │(Interrupt)│
└────────┘    └────────┘    └────────┘    └────────┘
                  ▲                            │
                  └────────────────────────────┘
                      (有中断请求时进入中断周期)
```

> **注意**：并非所有指令都需要完整的四个阶段。例如有些指令取指后直接可执行，不需要译码周期（如果采用硬布线控制器则取指和译码可以重叠）；中断周期只在有中断请求时才发生。

### 5.2.2 取指周期（Fetch Cycle）

取指周期完成从主存中取出一条指令并完成PC更新：

| 步骤 | 微操作 |
|------|--------|
| 1 | `MAR ← (PC)` — 将PC内容送地址寄存器 |
| 2 | `MDR ← M(MAR)` — 从主存取指令到MDR |
| 3 | `IR ← (MDR)` — 将指令送入指令寄存器 |
| 4 | `PC ← (PC) + 1` — PC指向下一条指令 |

> 其中 `M(MAR)` 表示以MAR内容为地址的主存单元的内容。"+1"中的"1"实际为一个指令字的字节数。

### 5.2.3 间址周期（Indirect Cycle）

如果指令使用了间接寻址，需要额外的间址周期来获取操作数的真实地址：

| 步骤 | 微操作 |
|------|--------|
| 1 | `MAR ← A`（IR中的形式地址部分） |
| 2 | `MDR ← M(MAR)` |
| 3 | `MAR ← (MDR)`（得到有效地址） |

### 5.2.4 执行周期（Execute Cycle）

不同指令的执行周期微操作序列不同。以 `ADD A`（加操作）为例：

| 步骤 | 微操作 |
|------|--------|
| 1 | `MAR ← A`（或由间址周期已完成） |
| 2 | `MDR ← M(MAR)` — 从主存取操作数 |
| 3 | `ACC ← (ACC) + (MDR)` — ALU执行加法 |

### 5.2.5 中断周期（Interrupt Cycle）

| 步骤 | 微操作 |
|------|--------|
| 1 | `SP ← (SP) - 1; M(SP) ← (PC)` — 保存断点PC |
| 2 | `SP ← (SP) - 1; M(SP) ← (PSW)` — 保存PSW |
| 3 | `PC ← 中断向量地址` — 转入中断服务程序 |

## 5.3 数据通路（Data Path）⭐

### 5.3.1 数据通路的概念

数据通路是指**数据在CPU内部各部件之间传输的路径**。它是运算器和控制器协同工作的核心。数据通路的设计决定了CPU能够在一个时钟周期内完成哪些操作。

### 5.3.2 单总线结构的数据通路

```
   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐
   │ PC  │   │ MAR │   │ MDR │   │ IR  │   │ ACC │
   └──┬──┘   └──┬──┘   └──┬──┘   └──┬──┘   └──┬──┘
      │         │         │         │         │
      └─────────┴────┬────┴─────────┴─────────┘
                     │
              ┌──────┴──────┐
              │  内部总线     │
              └──────┬──────┘
                     │
           ┌─────────┼─────────┐
           │         │         │
        ┌──┴──┐   ┌──┴──┐   ┌──┴──┐
        │ ALU │   │  Y  │   │  Z  │
        └─────┘   └─────┘   └─────┘
                  (暂存器)   (暂存器)
```

> **单总线结构**：所有寄存器都连接到同一条内部总线上。同一时刻只能有一条数据在总线上传输，因此完成一条指令需要多个时钟周期。结构简单但效率较低。

### 5.3.3 多总线结构的数据通路

```
   ┌─────────────── 地址总线 ───────────────┐
   │                                        │
┌──┴──┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌──┴──┐
│ PC  │   │ MAR │   │ IR  │   │ ACC │   │ R0  │ ...
└──┬──┘   └──┬──┘   └──┬──┘   └──┬──┘   └──┬──┘
   │         │         │         │         │
   └─────────┴────┬────┴─────────┴─────────┘
                  │
         ┌────────┼────────┐
         │        │        │
    ┌────┴──┐ ┌───┴───┐ ┌──┴─────┐
    │ 总线1  │ │ 总线2  │ │ 总线3  │  ← 多组独立总线
    └────┬──┘ └───┬───┘ └──┬─────┘
         │        │        │
      ┌──┴──┐  ┌──┴──┐  ┌──┴──┐
      │ ALU │  │ MDR │  │寄存器│
      └─────┘  └─────┘  │ 堆  │
                        └─────┘
```

- **双总线结构**：一条专门用于ALU操作，另一条用于寄存器间传输
- **三总线结构**：两条输入总线（ALU的两个操作数）+ 一条输出总线，允许在一个时钟周期内完成一次ALU操作

### 5.3.4 专用数据通路

不采用共享总线，而是各部件之间用专门的连接线相连。速度最快但成本最高、灵活性最差。

```
示例：ADD R1, R2 的专用数据通路操作

R1 ──▶──── ALU输入1 ──┐
                      ├──▶ ALU ──▶ R1
R2 ──▶──── ALU输入2 ──┘
```

## 5.4 控制器（CU）的设计 ⭐⭐⭐

控制器是CPU的"大脑"，负责产生指令执行所需的所有控制信号。它的实现方式主要有两种：**硬布线控制器**和**微程序控制器**。

### 5.4.1 控制器的功能与信号

控制器的输入和输出：

```
                    输入
                     │
    ┌────────────────┼────────────────┐
    │ 时钟信号        │                │
    │ 指令寄存器(IR)  │                │
    │ 状态标志(PSW)   │                │
    │ 外部请求(中断等) │                │
    └────────────────┼────────────────┘
                     ▼
              ┌─────────────┐
              │   控制器CU   │
              └──────┬──────┘
                     ▼
    ┌────────────────┼────────────────┐
    │ ALU操作选择信号 │ 寄存器选择信号  │
    │ 存储器读写信号  │ 总线控制信号    │
    │ PC控制信号     │ ...           │
    └────────────────┼────────────────┘
                     │
                   输出（控制信号）
```

### 5.4.2 硬布线控制器（Hardwired Control）

#### 基本原理

硬布线控制器使用**组合逻辑电路**直接产生控制信号。每个控制信号都是输入的布尔函数：

$$C_i = f(OP, \text{时钟}, \text{标志位}, \dots)$$

#### 设计步骤

```
1. 列出指令的操作时间表
2. 写出每个控制信号的逻辑表达式
3. 化简逻辑表达式
4. 用门电路实现
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 速度快（门电路延迟极小） | 设计复杂、不易修改 |
| 适合RISC处理器 | 指令增加时电路复杂度急剧上升 |
| 状态少，时序简单 | 不适合复杂指令集 |

> **RISC处理器几乎都采用硬布线控制器**，因为指令简单、格式规整，组合逻辑实现最为高效。

### 5.4.3 微程序控制器（Microprogrammed Control）⭐

#### 基本原理

将每条机器指令分解为一系列**微操作**，每条微操作对应一条**微指令**，若干条微指令组成一段**微程序**。微程序存放在CPU内部的**控制存储器（CM, Control Memory）**中。

```
机器指令 → 微程序（一段微指令序列）→ 微操作 → 控制信号

    ┌──────────┐
    │ 指令 (IR) │
    └────┬─────┘
         ▼
    ┌────────────────┐
    │ 微地址形成电路   │  ← 根据操作码生成微程序入口地址
    └────┬───────────┘
         ▼
    ┌────────────────┐
    │ 控制存储器(CM)  │  ← 存放所有微程序
    │   (ROM实现)     │
    └────┬───────────┘
         ▼
    ┌────────────────┐
    │  微指令寄存器   │
    │  (μIR)         │
    └────┬───────────┘
         ▼
    ┌────────────────┐
    │  控制信号输出   │
    └────────────────┘
```

#### 微指令的格式

```
┌──────────────┬─────────────────┐
│  操作控制字段  │  顺序控制字段    │
│ (产生控制信号) │ (决定下一条微指令) │
└──────────────┴─────────────────┘
```

#### 微指令的编码方式

| 方式 | 原理 | 特点 |
|------|------|------|
| **直接编码（不译法）** | 每一位直接控制一个微操作 | 速度快但微指令很长 |
| **字段直接编码（显式编码）** | 互斥的微操作分在同一字段，经译码产生控制信号 | 折中方案，最常用 |
| **字段间接编码（隐式编码）** | 一个字段的编码含义由另一个字段决定 | 最紧凑但最复杂 |
| **混合编码** | 以上方式结合 | 现代CPU常用 |

#### 微程序控制器的优缺点

| 优点 | 缺点 |
|------|------|
| 设计规整、易于修改 | 速度慢（每次都要读控制存储器） |
| 适合复杂指令集（CISC） | 增加了一级存储访问延迟 |
| 便于调试和扩展 | 控制器面积大 |

#### 微程序控制器 vs 硬布线控制器

| 对比维度 | 硬布线控制器 | 微程序控制器 |
|----------|------------|-------------|
| 实现方式 | 组合逻辑电路 | 存储逻辑（微程序+CM） |
| 速度 | **快** | 慢 |
| 灵活性 | 差（修改需重新布线） | **好**（改微程序即可） |
| 规整性 | 差 | **好** |
| 成本 | 低（简单指令集时） | 需要控制存储器 |
| 适用 | RISC | CISC |
| 代表 | ARM、MIPS、RISC-V | 早期x86、VAX |

> **现代趋势**：大多数现代处理器采用硬布线控制器，因为速度是首要考虑因素。即使是CISC的x86，也在内部将复杂指令翻译为简单微操作后，用类似硬布线的方式执行。

## 5.5 指令流水线的初步概念

> 注：流水线技术将在第7章详细展开，本节仅从CPU设计的角度介绍基本概念。

### 5.5.1 流水线的思想

将指令执行过程划分为多个阶段，每个阶段使用独立的硬件部件，从而允许多条指令重叠执行。

```
经典五级流水线：

IF ──▶ ID ──▶ EX ──▶ MEM ──▶ WB
取指    译码    执行    访存    写回
```

| 时钟周期 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 |
|----------|---|---|---|---|---|---|---|---|---|
| 指令i | IF | ID | EX | MEM | WB | | | | |
| 指令i+1 | | IF | ID | EX | MEM | WB | | | |
| 指令i+2 | | | IF | ID | EX | MEM | WB | | |
| 指令i+3 | | | | IF | ID | EX | MEM | WB | |
| 指令i+4 | | | | | IF | ID | EX | MEM | WB |

> 理想情况下，流水线每个时钟周期可以**完成一条指令**（而单周期方式每个时钟周期最多开始一条）。

### 5.5.2 流水线的性能指标

- **吞吐率（Throughput）**：单位时间内完成的指令数
- **加速比（Speedup）**：非流水线执行时间 / 流水线执行时间
- **效率（Efficiency）**：流水线各段的利用率

## 5.6 异常与中断

### 5.6.1 基本概念

| 概念 | 含义 | 实例 |
|------|------|------|
| **异常（Exception）** | CPU内部产生的意外事件，与指令执行同步 | 除零、非法指令、缺页 |
| **中断（Interrupt）** | CPU外部设备发来的信号，与指令执行异步 | 键盘输入、网卡收包、定时器 |

### 5.6.2 中断响应流程

```
1. 关中断（禁止响应其他中断）
2. 保存断点和PSW
3. 识别中断源，获取中断向量
4. 转入中断服务程序
5. 执行中断服务程序
6. 恢复现场（恢复PSW和断点）
7. 开中断
8. 返回被中断的程序
```

### 5.6.3 中断屏蔽

| 概念 | 说明 |
|------|------|
| **可屏蔽中断（INTR）** | 可通过设置中断允许标志（IF）来决定是否响应 |
| **不可屏蔽中断（NMI）** | 必须立即响应，如电源掉电、硬件故障 |

## 本章小结

CPU章是计算机组成原理的核心章节，知识体系总结：

```
                    CPU
                     │
     ┌───────────────┼───────────────┐
     ▼               ▼               ▼
  运算器(ALU)     控制器(CU)      寄存器组
     │               │               │
     │        ┌──────┴──────┐        │
     │        ▼             ▼        │
     │   硬布线控制器   微程序控制器   │
     │   (组合逻辑)   (存储逻辑)     │
     │                              │
     └────────────┬─────────────────┘
                  ▼
            数据通路设计
       (单总线/多总线/专用通路)
                  │
                  ▼
            指令周期
    (取指→间址→执行→中断)
```

**最重要的核心概念**：
1. **指令周期**是CPU工作的基本节拍，理解取指/间址/执行/中断四个阶段
2. **数据通路**决定了CPU的性能上限，多总线结构是现代CPU的主流
3. **硬布线 vs 微程序**是控制器设计的两种哲学，RISC选硬布线追求速度，CISC也曾用微程序追求灵活性
4. **异常和中断**是CPU与外部世界交互的重要机制
